# Monorepo 策略与未来应用

Readio 正在从单一前端代码库演进为 Cloud 托管 Web App 加未来平台入口的形态。
本文档概述 `apps/cloud-ui`、`apps/cloud-api` 的当前角色，以及 `apps/native` 的长期方向。

## 1. 应用概览

| 应用 | 路径 | 类型 | 状态 | 目标 |
|---|---|---|---|---|
| **Cloud UI** | `apps/cloud-ui` | SPA/Web App | 🟢 Live | 承载播放、discovery、library、files 与 settings 的主要前端表面。 |
| **Cloud API** | `apps/cloud-api` | Go 后端 | 🟢 Live | 提供 `apps/cloud-ui/dist`、同源 relay/proxy/fallback 路由，以及 SQLite 启动。 |
| **Native** | `apps/native` | Mobile | ⚪ 仅脚手架 | 通过 Capacitor 的 iOS/Android 包装器。 |

---

## 2. Cloud 策略 (后端)

**角色**: 可选增强。App 必须在没有它的情况下完美运行。

### 前后端拆分
- **当前状态**: `apps/cloud-api` 是当前 Go `net/http` 后端，直接提供构建后的 `apps/cloud-ui/dist`，并拥有同源 backend relay。
- **当前前端**: `apps/cloud-ui` 是当前 Web 前端，不再是“临时 Lite 产物”的描述。
- **未来方向**: 如果 Cloud 后续演变为更完整的多用户 Web 产品，再引入新的共享包或新的前端拆分。现状文档必须只描述当前真实代码。

### 计划功能 (P2/P3)
- **数据同步**: 跨设备同步订阅和进度。
- **AI 服务**: 服务端 Whisper (转录) 和 LLM (摘要)。
- **社交**: 评论和“播客俱乐部”。

### 技术栈
- **框架**: 当前脚手架使用 Go `net/http`。
- **数据库**: 当前脚手架使用 SQLite；PostgreSQL 仅保留为未来 SaaS 方向。
- **认证**: JWT 认证仅保留为未来方向。

---

## 3. Native 策略 (包装器)

**角色**: 访问 PWA 无法触及的系统 API。

### 计划功能 (P2)
- **后台音频**: 原生 iOS/Android 音频会话处理。
- **通知**: 新剧集的推送通知。
- **小组件**: 主屏幕小组件。
- **CarPlay / Auto**: 驾驶时的简化 UI。

### 技术栈
- **框架**: 未来仍可采用 Capacitor 作为包装层。
- **共享代码**: 当前不要假设不存在的共享包。

---

## 4. 共享包 (`packages/`)

现状文档只应描述当前真实存在、真实被使用的共享包。

- 不要继续把 `packages/core` 视为现行契约，除非该包重新存在并被工作区使用。
- 只有在共享代码真正抽取出来之后，才在此处声明新的共享包拓扑。
 
---
 
## 5. 引擎策略 (Engine Policy)

为了防止架构漂移和“在我的机器上能运行”这类 Bug，整个 monorepo 严格强制执行工具版本：

- **运行时 (Runtime)**: Node.js `>=20.0.0` (LTS 基准)。
- **包管理器 (Manager)**: pnpm `>=10.28.0` (Lockfile 版本稳定性)。
- **强制措施**: 在根目录 `.npmrc` 中强制设置 `engine-strict=true`。

---

## 6. 缓存策略 (Turborepo) [已实现]

为了最大化构建性能并防止过时的输出：

- **全局依赖**: `.npmrc` 和 `pnpm-workspace.yaml` 变更时会使所有缓存失效。
- **构建输入**: 源文件 (`src/**`)、配置文件 (`vite.config.ts`、`tsconfig.json`)，排除测试文件。
- **构建输出**: `dist/**`、`.next/**`、`stats.html`。
- **依赖传播**: `^build` 确保上游包先重新构建。
- **不缓存任务**: `dev` 和 `format` 永远不缓存。

**验证**: 运行 `pnpm build` 两次。第二次应显示 `>>> FULL TURBO`。
