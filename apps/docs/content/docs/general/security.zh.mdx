# 安全与隐私标准

Readio 致力于用户数据主权和绝对安全。

---

## 1. HTML 净化 (反 XSS)

**规则**: 绝不要直接使用 `dangerouslySetInnerHTML` 处理来自 RSS 源或文件的原始数据。
- **严格要求**: 所有 HTML 内容在渲染前必须经过净化。
- **工具**: 使用 `DOMPurify` (通过包装器 `src/lib/htmlUtils.ts`)。
- **策略**: 仅允许安全标签 (格式化, 链接)。脚本、iframe 和跟踪像素会被剥离。

## 1.1 内容安全策略 (CSP) [已实现]
- **状态**：通过指令 041 已实现。
- **目标**：通过 **服务器/边缘层的 HTTP 响应头** 强制严格 CSP，以限制脚本、媒体与图片来源。
- **策略**：`default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: blob: https:; media-src 'self' blob: https:; connect-src 'self' https:; font-src 'self' data: https:; frame-ancestors 'none'; base-uri 'self';`
- **灵活性**：显式允许 `connect-src` 和 `img-src` 使用 `https:` 协议，以支持用户配置的自定义 CORS 代理以及各种播客封面 CDN。
- **落地**：
  - **Vercel**：在 `vercel.json` 中通过响应头设置 `Content-Security-Policy`。
  - **Docker (Nginx)**：在 `nginx.conf` 中通过 `add_header ... always;` 设置 `Content-Security-Policy`。
  - **禁止**：不要通过 `index.html` 的 `<meta http-equiv="Content-Security-Policy">` 强制 CSP；它不是等价的控制面（尤其是 `frame-ancestors`）。

## 1.2 计划中：Iframe 沙箱
- **状态**：指令 051 计划中。
- **目标**：在需要时用沙箱 iframe 隔离不可信富文本。

## 2. 隐私与数据主权

### A. 代理隐私 (关键)
**问题**: 第三方 CORS 代理会看到你请求的每个 URL。
**规则**: 
- **用户选择**: 在设置中提供一个开关来禁用公共代理。
- **披露**: 当请求通过公共代理路由时，在 UI 中警告用户。
- **敏感数据**: 切勿通过公共代理传递身份验证标头、API 密钥或 Cookie。

### A.1. 公共代理安全加固 [已实现]
**状态**：通过指令 084 已实现。
**安全模型**：
- **前端配置是公开的**：任何打包到浏览器运行配置的值（`READIO_CORS_PROXY_AUTH_VALUE`）**不是**秘密存储。如需真正的秘密保护，请将秘密移至服务端代理。
- **加固策略**：将代理凭证视为非秘密，通过严格的请求约束、滥用控制和审计日志加固代理路径。

**代理请求约束** (Cloudflare Worker 端)：
- **允许上游主机列表**：仅批准的域名（iTunes、RSS 主机）。
- **允许调用方来源列表**：显式 `ALLOWED_ORIGINS`（逗号分隔）。开发阶段可临时包含 `http://localhost:5173`；**生产环境绝不能包含任何 localhost 来源**。
- **允许方法列表**：仅 `OPTIONS`（预检）和 `POST`（客户端协议）。
- **认证头协议**：客户端认证仅通过请求头携带（如 `x-proxy-token`），绝不通过查询参数。
- **仅 POST 客户端**：所有客户端代理调用使用 POST JSON body `{ "url": "..." }`。无 GET 回退。
- **响应策略**：对策略违规（不允许的来源/上游）返回 `403`。

**Lite 端防护**：
- 强制自定义代理模式为 POST JSON 协议；无运行时 GET 模式。
- 将认证头固定为 `x-proxy-token`，与 Worker 的 CORS allow-headers 保持一致，避免预检不匹配。
- 验证配置认证头时认证值必须存在。
- 保持 `credentials: 'omit'`。

### B. 错误报告隐私
**规则**: 不要收集个人身份信息 (PII)。
- **禁止**: 不要记录文件名、本地文件路径、播客标题或用户查询。
- **脱敏**: 诊断日志在写入缓冲区或导出前必须经过 `redact()` 过滤。
- **允许**: 通用错误信息/代码、浏览器版本和匿名堆栈跟踪（必须从路径中剥离本地用户名）。

## 3. 外部链接
**规则**: 使用 `src/lib/openExternal.ts` 进行所有外部导航。
- **要求**: 必须包含 `rel="noopener noreferrer"`。

## 4. 本地文件安全
**规则**: App 绝不能尝试将本地媒体文件的内容上传到任何服务器。
- **处理**: 所有元数据提取 (ID3 标签) 必须在客户端发生 (通过 Web Workers)。

## 5. 依赖项安全与供应链 [已实现]

**目标**：保障供应链安全，确保所有依赖项都是安全的，且符合项目的开源许可证政策。

- **状态**：通过指令 078 已实现。
- **漏洞审计**：CI 流水线在每个 PR 上运行 `pnpm audit --audit-level high`，**报告模式（不阻断）**。高危或严重漏洞会在日志中报告。
- **许可证合规性**：CI 流水线包含 **许可证检查** 步骤（通过 `license-checker-rseidelsohn`），**报告模式（不阻断）**，用于审计第三方许可证。
- **政策**：生产依赖项仅允许使用宽松协议（MIT、Apache-2.0、BSD、ISC）。禁止使用传染性协议（GPL、LGPL、AGPL）。
