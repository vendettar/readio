# 可访问性 (A11y) 标准

Readio 必须对每个人都可用。
作为一个 "Vibe Coding" 项目，我们依赖 **Headless UI** 库 (Radix/Shadcn) 来处理繁重的工作，但我们必须正确使用它们。

## 1. “无鼠标”规则

**规则**: 每个功能必须**仅用键盘**可用。
- **Tab**: 在交互元素之间移动焦点。
- **Enter/Space**: 激活按钮/链接。
- **Esc**: 关闭模态/菜单。

**自检**: 在完成任务之前，收起鼠标并尝试使用该功能。

## 2. 交互元素

### 按钮 vs Divs
- ❌ **错误**: `<div onClick={...}>` (对键盘/屏幕阅读器不可见)
- ✅ **正确**: `<Button>` 或 `<div role="button" tabIndex={0} onKeyDown={...}>`
- **Vibe 规则**: 只需使用 `shadcn/ui` 组件 (`Button`, `Card` 作为按钮)。它们为你处理此问题。

### 图标按钮
每个仅图标按钮必须有一个标签。

```tsx
// ❌ 坏
<Button><PlayIcon /></Button>

// ✅ 好 (视觉隐藏标签)
<Button size="icon" aria-label="Play Episode">
  <PlayIcon />
</Button>
```

## 3. 焦点管理

- **焦点环**: 永远不要移除默认焦点环 (`outline-none`)，除非你用自定义 `ring-*` 工具类替换它。用户必须知道他们在哪里。
- **模态**: 打开 Dialog 时，焦点必须移入内部。关闭时，焦点必须返回到触发器。（Radix 会处理这个问题，不要破坏它）。

## 4. 语义 HTML

不要只用 `<div>` 做所有事情。
- 列表: `<ul>` / `<ol>`
- 标题: `<h1>` -> `<h2>` -> `<h3>` (层级很重要)
- 文章: `<article>` 用于播客卡片。

## 5. 颜色对比度

- 确保文本在背景下可读。
- 小心使用 Tailwind 的 `text-muted-foreground`——确保它不会太微弱。
