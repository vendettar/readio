# Architecture Decision Log

## D001 — UUID Primary Keys (All Tables)
- **Decision**: All tables use UUID primary keys; natural keys (e.g., `feedUrl`, `key`) are unique indexes only.
- **Rationale**: Global uniqueness, sync readiness, stable identity.
- **Implication**: Schema changes require DB reset (first‑release policy).

## D002 — I18n Direct Usage
- **Decision**: Components use `useTranslation()` from `react-i18next` directly; no `useI18n` adapter.
- **Rationale**: Standard API, fewer layers, clearer onboarding.

## D003 — Offline Strategy A
- **Decision**: When offline, hide remote Discovery and remote search results; keep local content available and show an Offline badge.
- **Rationale**: Predictable UX, no failing network calls.

## D004 — ObjectURL Ownership
- **Decision**: Audio Object URLs are store‑owned; image Object URLs are component‑owned with cleanup on unmount.
- **Rationale**: Prevent audio memory leaks while keeping UI flexibility for artwork.

## D005 — Deployment Target [Implemented]
- **Decision**: Primary deployment is Vercel; Docker is supported for self‑hosting.
- **Rationale**: Fast web deployment with a portable fallback.

## D006 — Legal Pages Required [Implemented]
- **Decision**: Privacy Policy and Terms of Service are mandatory for production.
- **Rationale**: Store compliance and user trust.

## D007 — Smoke E2E Tests [Implemented]
- **Decision**: Maintain Playwright smoke tests for core user journeys before release.
- **Rationale**: Prevent end‑to‑end regressions that unit tests miss.

## D008 — Strict CSP [Implemented]
- **Decision**: Enforce a strict Content Security Policy for all web deployments, via **HTTP response headers** (server/edge).
- **Rationale**: Reduce XSS and untrusted resource loading risk.

## D009 — Data Retention Policy [Implemented]
- **Decision**: Prune playback history to a bounded window (time + count).
- **Rationale**: Prevent long‑term IndexedDB bloat and performance degradation.

## D010 — PWA Update Lifecycle [Implemented]
- **Decision**: Notify users when a new PWA version is available and allow refresh.
- **Rationale**: Avoid stale clients and ensure users receive fixes.

## D011 — CI Pipeline [Implemented]
- **Decision**: Enforce lint/typecheck/tests on every PR via CI.
- **Rationale**: Prevent regressions from bypassing local checks.

## D012 — Import Integrity Checks [Implemented]
- **Decision**: Validate imports for ID collisions and broken references before writing to DB (Instruction 049).
- **Rationale**: Prevent silent corruption during Vault/OPML import.

## D013 — Bundle Size Budget [Implemented]
- **Decision**: Enforce bundle size budgets in CI (Instruction 050).
- **Rationale**: Prevent performance regressions from dependency bloat.

## D014 — Iframe Sandbox Policy
- **Decision**: Use strict iframe sandboxing for untrusted rich HTML when required.
- **Rationale**: Isolate untrusted content beyond DOMPurify.

## D015 — Feedback Loop
- **Decision**: Provide in-app links for bug reporting and support contact.
- **Rationale**: Close product feedback loop.

## D016 — Offline Content Visibility
- **Decision**: Show offline availability badges and provide cache cleanup tools.
- **Rationale**: Make offline behavior transparent and manageable.

## D017 — Episode Sorting & Filtering
- **Decision**: Provide sort/filter controls for episode lists.
- **Rationale**: Improve discoverability at scale.

## D018 — Content Attribution
- **Decision**: Show attribution and copyright complaint entry points.
- **Rationale**: Reduce legal exposure and clarify content ownership.

## D019 — Multilingual Typography [Implemented]
- **Decision**: Use language-specific font stacks via `:root[lang]` selectors.
- **Rationale**: Ensure consistent rendering across scripts (Latin, CJK).
- **Implementation**: Language-specific font stacks in `index.css` with optimizeLegibility rendering.

## D020 — DB Access Guard [Implemented]
- **Decision**: Prevent direct DB access from UI components via lint guard.
- **Rationale**: Enforce architecture boundaries and maintain store-first persistence.
- **Implementation**: `pnpm lint:db-guard` script checks for DB imports in components.

## D021 — MSW for Network Tests [Implemented]
- **Decision**: Use MSW to mock external APIs in tests.
- **Rationale**: Stable and deterministic test runs.
- **Implementation**: `src/__tests__/setup.ts` and `handlers.ts` intercept requests.

## D022 — Contributing & Manifesto
- **Decision**: Document instruction-driven workflow and architecture manifesto.
- **Rationale**: Reduce onboarding friction and keep rules consistent.

## D023 — PWA Audio Range Cache [Implemented]
- **Decision**: Support Range Requests (206) in SW media cache for offline seeking.
- **Rationale**: Enable offline seek in large audio files.

## D024 — Memory Stress Test [Implemented]
- **Decision**: Maintain a long-session memory stress test for leak detection.
- **Rationale**: Prevent mobile crashes after hours of playback.
- **Implementation**: `apps/lite/tests/stress/memory.test.ts` (Playwright).

## D025 — Proxy Failover
- **Decision**: Support proxy list with automatic failover on 429/5xx.
- **Rationale**: Keep discovery resilient when a proxy fails.

## D026 — I18n Coverage Audit [Implemented]
- **Decision**: Enforce translation key parity across all languages at build time.
- **Rationale**: Prevent missing-key regressions in non-English locales.
- **Implementation**: `apps/lite/scripts/audit-i18n.ts` compares all languages against `en` and is integrated into CI (Instruction 063).

## D027 — IndexedDB Migrations
- **Decision**: Move from reset-only policy to versioned migrations.
- **Rationale**: Preserve user data in production.

## D028 — Browser Support Matrix
- **Decision**: Define browserslist and legacy polyfill strategy.
- **Rationale**: Avoid white screens on older devices/WebViews.

## D029 — Turborepo Cache Hardening [Implemented]
- **Decision**: Explicit inputs/outputs for Turbo pipelines.
- **Rationale**: Ensure cache correctness and fast CI.
- **Implementation**: `turbo.json` defines explicit `inputs` (source files, configs) and `outputs` (`dist/**`, `stats.html`). Global dependencies (`.npmrc`, `pnpm-workspace.yaml`) are tracked. Test files are excluded from build inputs (Instruction 066).

## D030 — Atomic Write & Self-Healing [Implemented]
- **Decision**: Complex writes (imports, deletes) must be transactional; boot process checks integrity.
- **Rationale**: Prevent "half-written" state if browser crashes during large file ingestion.
- **Implementation**: Wrapped `ingestFiles` and `deleteFolder` in `db.transaction`. Added `runIntegrityCheck` to `useAppInitialization` to flag orphan (blob-less) tracks (Instruction 067).

## D031 — Async Race Condition Guard [Implemented]
- **Decision**: Use requestId/AbortController to ensure last-action-wins.
- **Rationale**: Avoid playback/UI desync under rapid interactions.
- **Implementation**: Added `loadRequestId` to `playerStore` and `exploreStore`; implemented auto-abort in `fetchUtils.ts` for same-resource requests.
- **Rule Documentation**: `apps/docs/content/docs/apps/lite/coding-standards/logic-flow.mdx` (Section: Async Interaction Safety).

## D032 — Robust Metadata Parser (Phase 1)
- **Decision**: Use `music-metadata` for ID3/M4A parsing.
- **Rationale**: Replaces filename-based heuristics with binary stream parsing to extract high-quality metadata (Artwork, Title, Album, Artist, Duration).
- **Artwork Strategy**: Extract embedded covers, store as Blobs in Dexie, and serve via ephemeral `ObjectURL` in UI.
- **Implementation**: Runs in dedicated Web Worker (`src/workers/metadata.worker.ts`) to prevent blocking the main thread during large file imports. Buffer polyfill applied in `main.tsx`.

## D033 — Skeleton UI System [Implemented]
- **Decision**: Standardize shimmer-based skeletons for key surfaces (PodcastCard, EpisodeRow, ExploreHero).
- **Rationale**: Reduce layout shift and improve perceived performance during content loading.
- **Implementation**: Base `Skeleton` component in `ui/skeleton.tsx` using `animate-shimmer`. Specific skeletons mirror layout dimensions of content components.

## D034 — Overlay Layering Standardization
- **Decision**: Standardize Portal usage and z-index tokens without a global modal store.
- **Rationale**: Prevent stacking bugs while keeping local state intact.

## D035 — Media Cache Eviction
- **Decision**: Enforce LRU-based audio blob trimming with configurable limits.
- **Rationale**: Prevent uncontrolled storage growth.

## D036 — Network Backoff & Circuit Breaker [Implemented]
- **Decision**: Add exponential backoff and per-proxy circuit breaker.
- **Rationale**: Avoid retry storms and improve proxy resilience.
- **Implementation**: 
    - `withRetry` helper in `fetchUtils.ts` with exponential backoff (1s, 2s, 4s, 8s) and jitter.
    - `CircuitBreaker` state management to block failing proxies for 30s after 5 consecutive failures.
    - Automatic retry on 5xx or network-level errors; skip for 4xx.

## D037 — Storage Quota Monitoring
- **Decision**: Surface quota usage and warnings in Settings.
- **Rationale**: Give users visibility before storage exhaustion.

## D038 — Intl API Upgrade
- **Decision**: Use Intl APIs for dates, numbers, and relative time.
- **Rationale**: Improve localization correctness and reduce manual formatting bugs.

## D039 — Granular Error Boundaries [Implemented]
- **Decision**: Add component-level error boundaries for critical UI nodes.
- **Rationale**: Prevent single-component crashes from taking down pages.
- **Implementation**:
    - `ComponentErrorBoundary` class component handles `getDerivedStateFromError` and `componentDidCatch`.
    - Integrated with `logError` for diagnosis.
    - Wrapped `TranscriptView`, `EpisodeRow`, and `Sidebar` for localized fault isolation.
    - Added i18n-supported localized fallback UI with a retry mechanism. (Instruction 076)

## D040 — Lighthouse CI Budget
- **Decision**: Enforce Lighthouse score budgets in CI.
- **Rationale**: Prevent regressions in performance and accessibility.

## D041 — Dependency Audit & License Policy
- **Decision**: Enforce vulnerability and license checks in CI.
- **Rationale**: Reduce supply-chain and compliance risk.

## D042 — Floating Promise Audit
- **Decision**: Require explicit error handling for async store actions.
- **Rationale**: Prevent silent failures and inconsistent state.

## D043 — Event Listener Leak Audit
- **Decision**: Standardize global listener management via a shared hook.
- **Rationale**: Avoid memory leaks under long sessions.

## D044 — Zustand Selector Optimization
- **Decision**: Enforce atomic selectors and `useShallow` for multi-select.
- **Rationale**: Reduce unnecessary re-renders.

## D045 — RSS Content CSS Isolation
- **Decision**: Isolate show-notes CSS via scoped reset (Shadow DOM fallback).
- **Rationale**: Prevent untrusted HTML from polluting UI.

## D046 — File Validation Strategy
- **Decision**: Layered approach for performance and security.
  - **Web (Lite)**: MIME type and extension validation only (UX focused).
  - **Backend**: Deep content validation (Magic numbers, full parsing).
  - **Native**: Decoder-based validation or lightweight head sniffing.
- **Rationale**: Keep client-side logic fast and lightweight while deferring intensive checks to secure environments.

## D047 — UI Pattern Change Gate
- **Decision**: Every modification to `ui-patterns/*.mdx` that introduces a new standardized interaction (e.g., standard forms, file drop zones) MUST be logged in the Decision Log.
- **Rationale**: Ensure architecture consistency and maintain a source of truth for UI standardization across the project (Instructions 012-015).

## D048 — Standardized File Drop Zone
- **Decision**: All file upload surfaces MUST use the `FileDropZone` component based on `react-dropzone` with Zod-based validation.
- **Rationale**: Provides consistent visual feedback (Glassmorphism), centralized error handling, and type-safe validation across the application (Instruction 015).

## D049 — Premium Glass Visuals
- **Decision**: Core structural surfaces (Sidebar, Player Footer, Overlays) MUST use Premium Glass materials (`backdrop-blur-xl`, `backdrop-saturate-150`, and `bg-background/80` or `bg-popover/95`).
- **Rationale**: Achieves a high-quality, native-like aesthetic that adapts to content and theme while maintaining legibility (Instruction 016).

## D050 — List Row Interactions
- **Decision**: All list items and cards MUST use `bg-accent/50` for hover states and `border-border/50` for separation, with "Smart Divider" logic (Instruction 016).
- **Rationale**: Maintains visual consistency across different list types and creates a focused, floating effect for interactive rows.

## D051 — Modern Toast Notifications (Sonner)
- **Decision**: Replace Radix Toast with `sonner` for all notification feedback.
- **Rationale**: Sonner provides superior developer experience (imperative API), better visual stacking, built-in promise support, and native-feeling animations (Instruction 017).

## D052 — Command Palette Search (cmdk)
- **Decision**: Replace custom floating Search Overlay with an inline sidebar `CommandPalette` based on `cmdk`.
- **Rationale**: Provides consistent keyboard navigation, improved accessibility, and a modern "inline sidebar search experience" that works globally (Instruction 018).
- **Implementation Detail Update**: Keep inline sidebar input UX unchanged, but standardize result panel shell to portal-backed `Popover` (`PopoverAnchor` + `PopoverContent`) and align close-path control to store-driven `onOpenChange(false)` / `closeOverlay()`.

## D053 — Standardized Badge System
- **Decision**: All status indicators (Subscriptions, Favorites, History, Files, Listen Later) in lists and Command Palette MUST use a unified visual language: right-aligned, vertically centered, and `text-primary` color.
- **Rationale**: Ensures visual consistency and reduces cognitive load by establishing a predictable location and color for "Status" metadata across the application.

## D055 — Standardized Bits Animations
- **Decision**: Introduce a set of standardized, high-quality animation components (Bits) in `src/components/bits/`, starting with `AnimatedList` and `CountUp`.
- **Rationale**: Elevates the "vibe" and perceived quality of the application with fluid, premium motion patterns (Framer Motion) while ensuring reusability and architectural consistency (Instruction 021).
- **Implementation**:
    - `AnimatedList`: Staggered entrance animations for lists and grids.
    - `CountUp`: Smooth spring-based numeric transitions for statistics and usage values.

## D056 — Conditional "Last Played" Timestamp Update
- **Decision**: The `lastPlayedAt` timestamp in `PlaybackSession` is ONLY updated when `isPlaying` is true.
- **Rationale**: Prevents session restoration (e.g., reopening a tab) or manual seeking while paused from incorrectly bumping an item to the top of "Recent" lists. "Last Played" should semantically represent active consumption of content.
- **Implementation**:
    - Removed forced `Date.now()` in `DB.updatePlaybackSession`.
    - `playerStore` actions (`updateProgress`, `saveProgressNow`) conditionally include `lastPlayedAt` based on the current `isPlaying` state.

## D057 — Shared Element Transitions
- **Decision**: Use `layoutId` (Framer Motion) to synchronize persistent UI elements (especially artwork) between global UI states (MiniPlayer ↔ FullPlayer).
- **Rationale**: Maintains spatial continuity, reduces visual pop-in, and achieves a high-end "native" feel.
- **Constraint**: The `layoutId` MUST be unique within the same visible tree. Conditional rendering (e.g., `isDesktop`) must be used if multiple nodes for different layouts (Mobile/Desktop) share the same semantic ID tracking the same object.

## D058 — App Boot Sequence
- **Decision**: Show a full-screen `BootLoader` (Breathing Logo) until all core IndexedDB stores are hydrated and the playback session is restored.
- **Rationale**: Prevents "Hydration Flash" (empty states briefly flickering) and provides a premium, native-app feel during cold starts.
- **Constraints**: 
    - Loader must use `fixed inset-0 z-50 bg-background` to perfectly cover content.
    - Transitions must use `AnimatePresence` for a smooth fade-out.

## D059 — Shared Element Transition Scoping
- **Decision**: If the same logical object appears multiple times on a single page, the `layoutId` MUST be scoped with a unique prefix (e.g., `sectionId`) to avoid collision.
- **Status Update (Superseded in transport by D080)**: `fromLayoutPrefix` is still the transition metadata key, but it now travels via typed `location.state` only, not URL `search`.
- **Rationale**: Prevents Framer Motion collisions when duplicate content exists in different UI sections, while keeping transition metadata out of canonical/shareable URLs.

## D060 — Logical Properties Migration (RTL Support)
- **Decision**: Migrate all physical CSS properties (`left`, `right`, `ml`, `mr`, `pl`, `pr`, `text-left`, `text-right`, `border-l`, `border-r`) to their logical equivalents (`start`, `end`, `ms`, `me`, `ps`, `pe`, `text-start`, `text-end`, `border-s`, `border-e`).
- **Rationale**: Enables seamless support for Right-to-Left (RTL) languages (e.g., Arabic, Hebrew) without duplicating CSS rules or using complex overrides. Aligns with modern web standards and improves future accessibility.
- **Constraint**: Directional icons must be explicitly flipped using `className="rtl:rotate-180"` where appropriate.

## D061 — Store Persistence Repository Boundary (Phase 1) [Implemented]
- **Decision**: Introduce `LibraryRepository`, `PlaybackRepository`, and `FilesRepository` as the persistence boundary for `exploreStore`, `historyStore`, `playerStore`, and `filesStore`.
- **Rationale**: Decouple store action/state logic from direct Dexie access while preserving existing store APIs and runtime behavior.
- **Implementation**:
  - Migrated target stores to repository calls.
  - Added repository parity tests and repository-boundary store tests.
  - Kept Dexie schema/version and `DB` exports unchanged.

## D062 — Fetch/Runtime Config Internal Module Extraction (Phase 1) [Implemented]
- **Decision**: Keep `fetchUtils.ts` and `runtimeConfig.ts` as stable public facades, and extract internals into dedicated modules.
- **Rationale**: Reduce critical-file complexity and isolate policy-heavy logic without changing external contracts.
- **Implementation**:
  - Added `lib/networking/proxyUrl.ts`, `lib/networking/circuitBreaker.ts`, `lib/networking/timeouts.ts`.
  - Added `runtimeConfig.defaults.ts` and `runtimeConfig.schema.ts`.
  - Preserved proxy fallback order, breaker policy, timeout behavior, env-key contract, and fallback parsing semantics.

## D061 — Sleep Timer Architecture
- **Decision**: Centralize the sleep timer countdown ticker inside `useSleepTimerStore` (module-level interval), while `GlobalAudioController` remains responsible for reacting to end-of-episode events.
- **Rationale**: Guarantees a single ticker regardless of component mounting patterns, avoids duplicate timers, and keeps countdown state co-located with the store that owns it. The player should revert to a `paused` state when the timer expires.

## D062 — Tooltip Wrapping for Overlays
- **Decision**: Standardize wrapping `DropdownMenuTrigger` or `PopoverTrigger` with `Tooltip` for icon-only buttons that also provide transient state feedback (e.g., remaining sleep timer time). 
- **Rationale**: Adheres to the "Icon-only buttons MUST have a Tooltip" rule while allowing rich interaction on click.

## D063 — Image Fallback Strategy [Implemented]
- **Decision**: All artwork/images MUST use a standard fallback mechanism: themed skeleton (`bg-muted/50`) during loading, and a fallback icon (`Podcast` or `Disc`) on error. Transition between states MUST use a smooth fade. (Instruction 046)
- **Rationale**: Prevents "broken image" icons from degrading the premium aesthetic and ensures layout stability during content ingestion.

## D064 — Onboarding & Empty States [Implemented]
- **Decision**: All empty library/list views MUST use a standardized `EmptyState` pattern: centered orientation, high-quality icon, clear descriptive text, and a primary CTA button. (Instruction 045)
- **Rationale**: Reduces user abandonment by providing clear guidance on how to populate the application and ensuring the first-run experience is intentional.

## D065 — UseEventListener Ref Handling
- **Decision**: `useEventListener` MUST receive the `RefObject` itself, not `ref.current`. The hook is responsible for accessing `.current` inside its effect.
- **Rationale**: Ensures listeners are correctly attached even if the ref is initially null (P1 fix).

## D066 — Prose Isolation Strategy
- **Decision**: Use a wrapper `div` with `.prose-isolate` around content that needs style isolation, instead of applying it directly to the content container.
- **Rationale**: Prevents `all: revert` from stripping utility classes needed on the content container itself (e.g., `whitespace-pre-line`).

## D067 — Floating Promise Guard
- **Decision**: All "fire-and-forget" async calls in stores MUST use `.catch(logError)` or `void` operator.
- **Rationale**: Prevents unhandled promise rejections and ensures errors are logged (Instruction 079).

## D068 — Store Selector Enforcement
- **Decision**: All Zustand store usage MUST use atomic selectors.
- **Rationale**: Prevents massive unnecessary re-renders. Enforced via `lint:selectors` CI check.

## D069 — Tab Synchronization
- **Decision**: Use `BroadcastChannel` (with `localStorage` fallback) to coordinate playback across tabs. Only one tab plays at a time.
- **Rationale**: Prevents audio overlapping (Instruction 029).

## D070 — CORS Proxy Architecture & Troubleshooting [2.0 Refactor]
- **Decision**:
  - Standardize all CORS proxies to raw/passthrough mode (original `Content-Type` + body), and decommission AllOrigins-specific JSON wrapper assumptions.
  - Keep proxy configuration provider-agnostic via URL template (`{url}`), so Cloudflare Workers/Nginx reverse proxies can be swapped without client contract changes.
  - Disable circuit-breaker and global `abortPrevious` dedupe in client fetch fallback chains for discovery requests.
- **Rationale**:
  - Remove hard dependency on one public proxy service.
  - In single-user browser runtime, preemptive breaker/dedupe can drop valid later requests; exhaustive ordered retries across configured endpoints are more reliable for perceived availability.

## D071 — Library Detail Country Context (`countryAtSave`)
- **Decision**: Persist normalized `countryAtSave` on library/remote writes (`Subscription`, `Favorite`, podcast-origin `PlaybackSession`) and use it to build canonical `/$country/...` links from library surfaces.
- **Status Update (Superseded in resolution authority by D082 + D080)**: podcast content-route country resolution is route-param authority only (`$country`), with no runtime fallback to global/default country in content pages.
- **Write Guardrail**: if `countryAtSave` cannot be resolved for remote/library writes, reject persistence (local track sessions are exempt).
- **Rationale**: Region switching must affect discovery only; saved items keep deterministic country context through canonical path links under first-release, no-backfill policy.

## D072 — Discovery SWR Cache Semantics
- **Decision**: Discovery provider cache now uses SWR + stale-if-error semantics with explicit per-API TTL profiles (24h charts/detail/feed, 12h episode lookup list, 30m search), keeps stale localStorage payloads after TTL expiry, and only overwrites cache on successful network responses.
- **Rationale**: Preserve continuity on unstable networks and prevent empty-state regressions when stale data exists. Revalidation success must propagate to active UI via exact `queryClient.setQueryData`, while storage cleanup remains stale-safe (hard-retention protected) and explicit maintenance is centralized through `runDiscoveryCacheMaintenance()`.

## D073 — Podcast Transcript Auto-Ingestion Cache
- **Decision**: Podcast transcript ingestion is asynchronous and non-blocking to playback start. The system reads transcript data in order: runtime cache behavior (`fresh | stale | miss`) -> IndexedDB `remote_transcripts` -> network fetch, and writes cache only on successful fetch+parse.
- **Rationale**: Preserve read-along continuity without delaying audio controls, keep stale transcript usable during network failures, and avoid data loss from failed revalidation.
- **Guardrails**: Manual local subtitles stay authoritative; remote transcript apply must pass playback context guards (`loadRequestId` + `audioUrl`) to prevent stale async overrides.

## D074 — Transcript Atomic Lookup Authority & Span-Safe Highlight
- **Decision**:
  - Keep atomic transcript rendering (`SubtitleLine` tokenization + `Word` interactive tokens) with a single lookup authority per `TranscriptView` (`useSelection(containerRef)`).
  - Enforce event dedupe by marking word-token clicks with `data-lookup-word="true"` and skipping generic mouseup fallback lookup for marked targets.
  - Upgrade subtitle highlight matching to traverse all text nodes under `.subtitle-text` instead of assuming `element.firstChild` is a text node.
- **Rationale**: Eliminate duplicate lookup requests from one click, preserve deterministic selection/lookup state transitions, and keep dictionary-word highlight behavior correct after transcript spanization/nested DOM structures.

## D075 — Global Docked Transcript Surface Mode [Implemented]
- **Decision**:
  - Introduce a global player-surface state machine (`mini | docked | full`) in `src/store/playerSurfaceStore.ts`.
  - Playback with transcript-capable context defaults to `docked`; non-transcript context defaults to `mini`.
  - Surface transitions are route-independent and non-modal: collapse/restore/open-full must not trigger navigation.
  - Full-open is guarded by playable context; exiting full returns to `docked` when context exists, otherwise `mini`.
- **Rationale**: Keep transcript read-along available as a persistent player extension across routes while preserving shell interactivity (Sidebar/page content) and avoiding modal/route-coupled regressions.
- **Implementation**:
  - `AppShell` mounts `DockedPlayer` as the standard reading surface extension (no additional right-side panel).
  - `MiniPlayer` provides docked restore and full-open entry with guard semantics.
  - Playback entry paths (`useEpisodePlayback`, `useFilePlayback`, `HistoryPage`) set playable context and default surface deterministically.

## D076 — Component-Driven Transcript Word Interactions
- **Decision**:
  - `Word` component is the sole trigger source for single-word lookup and single-word context menu.
  - Add dedicated `openWordMenu(word, x, y)` action in `useSelectionActions` for word-menu state transitions.
  - Remove geometric word hit-testing/click-delay fallback lookup from `useSelectionEvents`; keep only selected-range menu routing and container-level close behavior.
  - Enforce context priority: when native selection is non-collapsed, selected-range context menu wins and word-menu dispatch is skipped for the same event.
- **Rationale**: Eliminate duplicate interaction paths under virtualization, reduce hot-path mouse event overhead, and keep selection/lookup state deterministic from a single `TranscriptView` authority.

## D077 — Transcript Right-Click Dual-Path Menu Contract
- **Decision**:
  - Transcript right-click keeps a deterministic dual-path contract:
    - selected text present -> open context menu in `menuMode: 'word'`
    - no selection on `.subtitle-line` -> fallback to line context menu in `menuMode: 'line'`
  - `Word` component remains the explicit source for single-word right-click menu (`openWordMenu`), and geometric pointer-based word lookup remains removed.
- **Rationale**: Preserve prior line-level UX while keeping component-driven word interactions and avoiding reintroduction of fragile geometric hit-testing paths.

## D078 — Unified Full/Docked Player Surface Motion [Implemented]
- **Decision**:
  - Replace separate `DockedPlayer`/`FullPlayer` overlays with one shared `PlayerSurfaceFrame` and one shared layout identity (`PLAYER_SURFACE_LAYOUT_ID`).
  - Keep docked/full as two layout states of the same logical reading surface and keep mode transitions route-independent.
  - Move transcript auto-follow state to shared frame scope so transcript interaction and follow-button visibility remain consistent across docked/full morph.
- **Rationale**: Remove the perceived two-step exit/enter sequence and enforce continuous single-surface motion while preserving transcript behavior and control parity.
- **Implementation**:
  - `AppShell` renders one surface frame topology for `docked | full`.
  - Shared reading core is extracted to `ReadingContent`.
  - Legacy `DockedPlayer.tsx` and `FullPlayer.tsx` are removed in favor of unified frame architecture (Instruction 089c).

## D079 — Full Surface Dismiss Interaction (Custom Motion) [Updated]
- **Decision**:
  - Replaced `Drawer` (`vaul`) with custom motion-based implementation for `PlayerSurfaceFrame`.
  - Full mode covers the entire screen (including sidebar) with a single unified surface shell.
  - Dismissal is explicit (Minimize button or `Esc` shortcut); backdrop click-dismiss is intentionally disabled.
- **Rationale**: `Drawer` caused layering issues with SideBar and unstable focus management. Custom motion provides a robust, fully controlled, and cleaner animation state that aligns with the "Mini -> Docked -> Full" morphing architecture.
- **Implementation**:
  - Removed `src/components/ui/drawer.tsx` and `vaul` dependency.
  - Removed backdrop click close path in `PlayerSurfaceFrame.tsx`; close path is now button/keyboard only and remains deterministic (`toDocked()` when restorable, else `toMini()`).

## D080 — Content URL Contract Finalization + Transition Metadata Relocation [Implemented]
- **Decision**:
  - All in-app content entry points (Explore/Search/Global Search/Library) must emit canonical path-only podcast routes with no correctness query hints.
  - `/$country/podcast/*` route search contracts no longer define legacy library hints (`source/feedUrl/audioUrl/providerEpisodeId/sessionId`) and no longer define `fromLayoutPrefix`.
  - Legacy query hints are tolerated and stripped during search validation (non-strict handling), rather than hard-rejected.
  - `fromLayoutPrefix` is retained only as optional shared-transition metadata in typed `location.state`, not URL `search`.
  - `useEpisodeResolution` no longer accepts context-based query hints; resolution correctness is route-param + slug-token driven.
- **Rationale**: Eliminate URL contract drift, prevent hidden route correctness coupling to search params, and keep shared-element transition metadata out of canonical links.

## D081 — Search Palette Opaque Selection Contrast [Implemented]
- **Decision**:

## D117 — Cloud ASR Provider Submission Uses a Dedicated Same-Origin Relay [Implemented]
- **Decision**:
  - Cloud ASR provider transcription submission must use a dedicated same-origin backend relay (`POST /api/v1/asr/transcriptions`) rather than browser-direct provider calls.
  - `/api/proxy` remains a bounded media/RSS proxy surface and must not become a generic provider POST tunnel.
  - In this phase, provider credentials remain user-owned and browser-supplied; the backend relay may use them only as transient request data and must not persist, cache, or log them.
- **Rationale**:
  - Browser-direct provider submission still fails under provider CORS even after the media fallback work is complete.
  - A dedicated relay fixes transport ownership without prematurely changing the product's user-provided-key model.
  - Keeping `/api/proxy` narrow avoids broadening SSRF/body-tunneling risk under a convenience abstraction.
- **Risk Notes**:
  - Provider verification/readiness remains browser-direct and is intentionally out of scope for this phase.
  - Browser-local blobs/downloaded blobs remain browser-local inputs; the relay does not move audio storage server-side.
  - Request bodies must stay bounded, and error mapping must preserve the existing ASR retry/cooldown semantics.

## D118 — Cloud Production Runtime Config Ownership Moved to Backend-Generated /env.js [Implemented]
- **Decision**:
  - Cloud production runtime config ownership moved to backend-generated `/env.js` by `apps/cloud-api`.
  - `apps/cloud-ui/public/env.js` is dev/default-template oriented, not production Cloud truth.
  - Server-owned runtime config (listen address, paths, abuse-control settings) belongs in VPS/systemd env files.
  - Browser-public runtime config is emitted by backend via an explicit allowlist, not dumped from process env.
- **Alternatives Considered**:
  - Keeping static frontend files (`dist/env.js`) as production truth: rejected because Cloud is a backend-owned product surface and production config should not require editing static frontend assets.
  - Shared config package: rejected as prematurely generalized; Cloud's ownership model differs from Lite's browser-direct model.
- **Rationale**:
  - Cloud now owns networking on the backend and must own the runtime configuration contract.
  - Production config should not require manually editing static frontend assets on the VPS.
  - Browser-public fields are not secrets; emitting them via backend allowlist makes the contract explicit and auditable.
- **Risk Notes**:
  - Browser-public fields (e.g., `READIO_ASR_RELAY_PUBLIC_TOKEN`) are lightweight abuse-control tokens, not strong secret boundaries.
  - If a field is emitted to `/env.js`, every browser user can read it; design accordingly.
  - The allowlist is implicit in the map literal construction in the Go handler (`apps/cloud-api/main.go`), and only browser-safe fields are emitted.
