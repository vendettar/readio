# Monorepo Strategy & Future Apps

Readio is designed to operate as a multi-app ecosystem.
This document outlines the current Cloud topology and the longer-term direction for `apps/native`.

## 1. Apps Overview

| App | Path | Type | Status | Goal |
|---|---|---|---|---|
| **Lite** | `apps/lite` | SPA/PWA | 🟢 Live | Zero-dependency, offline-first player. |
| **Cloud API (Current Scaffold)** | `apps/cloud-api` | Backend scaffold | ⚪ Scaffold only | Current Go scaffold: same-origin backend for Cloud web app, serving discovery/search/feed gateway, caching, and SQLite-backed services. |
| **Cloud API (Historical Scaffold Location)** | `apps/cloud` | Backend scaffold | ⚪ Historical | Previous location of the Cloud Go scaffold before the repository move to `apps/cloud-api`. |
| **Cloud UI (Current App)** | `apps/cloud-ui` | Web frontend | ⚪ Live | Cloud-only frontend that talks only to `apps/cloud-api` through same-origin APIs and does not expose Lite's CORS proxy model. |
| **Native** | `apps/native` | Mobile | ⚪ Scaffold only | iOS/Android wrapper via Capacitor. |

---

## 2. Cloud Strategy

**Role**: Optional enhancement. `apps/lite` must work perfectly without Cloud.

### Frontend Reuse Strategy (Critical Agent Architecture)
- **Current Topology**: `apps/cloud-api` is a pure Go `net/http` backend and `apps/cloud-ui` is the Cloud frontend. Cloud currently operates as two app boundaries:
  - `apps/cloud-api` for Go backend services
  - `apps/cloud-ui` for the Cloud frontend
- **Shared UI Rule**: `apps/lite` and `apps/cloud-ui` may share presentation components via `packages/ui`, but they must not share the same networking model.
- **Networking Boundary Rule**:
  - `apps/lite`: browser-backed networking, optional CORS proxy
  - `apps/cloud-ui`: same-origin backend-owned networking through `apps/cloud-api`
- **Do not** model Lite and Cloud as one web app with a build-time or runtime mode switch.

### Planned Features (P2/P3)
- **Data Sync**: Sync subscriptions and progress across devices.
- **AI Services**: Server-side Whisper (Transcription) and LLM (Summary).
- **Social**: Comments and "Podcast Clubs".

### Tech Stack
- **Framework**:
  - Current scaffold: Go `net/http` in `apps/cloud-api`
  - Current backend: Go `net/http` in `apps/cloud-api`
  - Current frontend: standalone web frontend in `apps/cloud-ui`
- **Database**: SQLite for the current scaffold; PostgreSQL remains a future SaaS direction only.
- **Auth**: JWT-based auth remains a future direction only.

## 3. Native Strategy (The Wrapper)

**Role**: Access system APIs that PWA cannot reach.

### Planned Features (P2)
- **Background Audio**: Native iOS/Android audio session handling.
- **Notifications**: Push notifications for new episodes.
- **Widgets**: Home screen widgets.
- **CarPlay / Auto**: Simplified UI for driving.

### Tech Stack
- **Framework**: Capacitor (wrapping `apps/lite`).
- **Shared Code**: Logic in `packages/core`.

---

## 4. Shared Packages (`packages/`)

To support this evolution, we must move only the correct kinds of logic out of `apps/lite` into shared packages.

- **`packages/core`**: Shared domain schemas and logic used by apps.
- **`packages/ui`**: Shared presentation components for `apps/lite` and `apps/cloud-ui`; this is a real shared presentation package and must stay networking-agnostic.
- **`packages/config`**: Shared tooling/config SSOT placeholder (directory exists; implementation pending).

Do not move Lite's browser-direct networking assumptions into shared packages.
 
---
 
## 5. Engine Policy (Consistency)

To prevent architectural drift and "Works on my machine" bugs, the entire monorepo strictly enforces tooling versions:

- **Runtime**: Node.js `>=20.0.0` (LTS baseline).
- **Manager**: pnpm `>=10.28.0` (Lockfile version stability).
- **Enforcement**: Mandatory `engine-strict=true` in root `.npmrc`.

---

## 6. Cache Policy (Turborepo) [Implemented]

To maximize build performance and prevent stale outputs:

- **Global Dependencies**: `.npmrc` and `pnpm-workspace.yaml` invalidate all caches when changed.
- **Build Inputs**: Source files (`src/**`), configs (`vite.config.ts`, `tsconfig.json`), excludes tests.
- **Build Outputs**: `dist/**`, `.next/**`, `stats.html`.
- **Dependency Propagation**: `^build` ensures upstream packages rebuild first.
- **No-Cache Tasks**: `dev` and `format` are never cached.

**Verification**: Run `pnpm build` twice. Second run should show `>>> FULL TURBO`.
