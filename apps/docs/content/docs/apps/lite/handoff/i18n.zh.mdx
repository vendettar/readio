---
title: 国际化与本地化 (i18n)
---

## 1. 架构设计

应用采用 **Typed Lazy Loading (类型化延迟加载)** 架构，平衡了类型安全与首屏加载性能。

- **存储位置**：`src/lib/locales/`。
- **模块划分**：每个语言拥有独立的 TypeScript 模块（如 `en.ts`, `zh.ts`）。
- **加载策略**：
  - **默认语言**：英文 (`en`) 作为基础 Schema 被首屏打包。
  - **按需加载**：非默认语言通过动态 `import()` 在用户切换时异步加载。

## 2. 安全切换契约

为了防止异步加载期间的竞态条件及界面“Key 泄露”，系统强制执行以下契约：

- **changeLanguageSafely**：外部组件仅允许通过该统一 API 触发语言变更。
- **并发去重**：同一语言的重复加载请求会被自动合并。
- **版本锁机制**：利用单调递增的请求版本号，确保只有用户最后一次选中的语言会被最终应用。
- **自愈回退**：若特定语言包加载失败，系统会自动回退至默认的英文资源并记录错误。

## 3. 类型安全与开发

- **基准 Schema**：以 `en.ts` 的结构作为类型定义的单一真理源。
- **自动补全**：利用 i18next 模块增强，`t()` 函数及 `TranslationKey` 类型具备完整的 IDE 自动补全支持。
- **同步校验**：通过 `audit-i18n.ts` 脚本自动化校验各语言包间的 Key 对齐情况。

## 4. 动态上下文

- **HTML Lang**：语言切换后会自动同步修改 `<html lang="...">` 属性，以适配不同语言的 CSS 字体栈。
- **格式化器**：日期、时长及数字格式化自动感应 `i18n.language` 状态。

## 5. 转录分词语言契约（Instruction 111）

- 转录分词现在将语言作为一等输入：`tokenizeWithLocale(text, language)`。
- 在分词器缓存查找前，会先对语言做规范化并兼容 `en-US` 这类 region 形式。
- 运行时做 `Intl.Segmenter` 特性检测；不支持时回退到确定性正则分词。
- 必须保留分隔符 token，确保标点与空白的渲染一致性。
