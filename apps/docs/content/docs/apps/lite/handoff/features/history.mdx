title: Playback History

## 1. Scope and Ownership

- Route: `/history`.
- Main Interface: `src/routeComponents/HistoryPage.tsx`.
- Data Authority: `src/store/historyStore.ts`.
- Main Table: IndexedDB `playback_sessions`.

## 2. Writing Contract

History records are automatically generated by the playback pipeline following these rules:

- **Session Persistence**:
  - Remote Podcasts: Creates or reuses sessions on playback start, recording `audioUrl`, `feedUrl`, and episode metadata.
  - Local Files: Marked as `source: 'local'`, recording `audioId` (Blob reference) and local track metadata.
- **Country Snapshot (SSOT)**: All remote records must include `countryAtSave` upon creation. Persistence is denied if this field cannot be resolved.
- **Heartbeat**: The `progress` field is updated every 5 seconds during active playback.

## 3. Reading and Interaction

- **List Loading**: Displayed in descending order of `lastPlayedAt`.
- **Artwork Lazy Loading**: Artwork for local files is retrieved asynchronously from `audioBlobs` to generate ObjectURLs.
- **Resuming Playback**:
  - Clicking a history item restores metadata to the Player Store while preserving the original Session ID.
- **Deletion**: Deleting history only removes the session record; the underlying local audio files are never deleted.

## 4. Navigation Standards

Jumping to episode details from history is supported under specific conditions:

- **Path Contract**: Must navigate to the canonical path `/$country/podcast/$id/episode/$episodeId`.
- **Route Parameters**: The country source strictly follows the `countryAtSave` stored in the record.
- **Fallback**: If jump metadata (like IDs) is missing, clicking the title falls back to direct playback.

## 5. Retention Policy

- **Auto-Pruning**: Prunes history at startup, keeping only the 1,000 most recent items or records within 6 months.
- **Integrity**: Periodic `runIntegrityCheck` ensures that sessions point to valid, non-corrupted audio references.

## 6. Overflow Stream-Only Action (Instruction 129)

- **Entry Point**: History row overflow now exposes `Play without transcript` only when:
  - session has a valid remote `audioUrl`
  - app is online
- **Behavior**:
  - delegates to shared `playHistorySessionWithDeps` with mode `stream_without_transcript`
  - if the episode is already downloaded, playback reuses local blob source and does not switch to remote stream URL
  - preserves existing surface policy transitions (`docked`/`mini` rules unchanged)
  - skips transcript auto-ingestion for that request
- **Guardrails**:
  - local-only history rows do not render this action
  - mode is request-scoped (later default plays still follow normal behavior)
