## 1. 范围与架构

搜索系统是应用全局的内容检索入口，采用两阶段工作流：

- **Command Palette (命令面板)**：侧边栏内嵌的快速入口 (`Cmd+K`)，基于 Popover 锚定架构。面板宽度为 `sidebar-width * 1.5`（约 390px，具备响应式收窄），最大高度为 `75vh`。
- **Search Page (搜索页)**：路由为 `/search?q=query` 的结果展示页。

## 2. 视觉与交互规范

### 列表项渲染
- **选中态**：活动项使用实心 `bg-primary` 背景。
- **边框合并 (Border Merging)**：条目使用 `-mt-px` 边距结合 `z-10` 层级，使其在视觉上覆盖前一项的分隔线，形成统一的选中区块。
- **图标标准化**：所有搜索/检索类的图标（如放大镜）必须统一使用 `h-4 w-4` 尺寸，以确保与输入框及其他列表项在视觉上精确对齐。
- **建议项去重**：命令面板顶部的快速建议项按标题 (`collectionName`) 执行去重逻辑，确保检索界面简洁高效。
- **高亮匹配策略**：采用多词项匹配模式。搜索词中的每个单词会独立匹配各处文字（非连续匹配），例如搜索 "the interview" 也会高亮 "The TED Interview" 中的对应单词。
- **状态徽章**：右对齐图标系统展示 Subscribed, Favorited, Played, Local 状态。多状态并列显示，元数据自动合并。

### 唯一导航原则
- 列表项禁止整行包装在 `Link` 中。
- 导航仅限标题 (`InteractiveTitle`) 和封面图 (`Button asChild`)。
- 描述区域不触发任何交互或跳转。
- 对于搜索单集行，封面上的圆形播放按钮始终为“直接播放”；标题与封面点击继续进入单集详情页。
- **Search 单集强约束**：在 Search Page 单集行中，标题/封面点击必须进入单集详情页 (`/$country/podcast/$id/episode/$episodeId`)，圆形播放按钮必须直接播放。

## 3. 性能与数据规范

### 数据库查询优化
- **严禁全表扫描**：所有搜索必须基于索引并配合节流 (Throttle)。
- **实时刷新**：结果组件直接订阅全局 Store 状态，确保在搜索过程中执行的收藏操作能即时反映在徽章上。
- **并发防护**：具备 200ms 防抖及 `requestId` 失效机制，防止旧请求回写导致的结果闪烁。

### 离线与鲁棒性
- **离线降级**：离线状态下自动隐藏远程建议，仅展示本地库匹配项。
- **异步补全**：若单集元数据缺失 `feedUrl`，收藏操作会自动触发异步查找逻辑以补全记录。
