## Multi-Media & OS Integration

### Media Session API (The OS Sync Rule)
**Rule**: The App's playback state MUST sync with the Operating System's media center.
- **Sync**: Title, Artist, and Artwork must be updated via `navigator.mediaSession.metadata`.
- **Control**: Play, Pause, Seek, and Skip actions must be mapped to `navigator.mediaSession.setActionHandler`.
- **Result**: Users should be able to control Readio from their Lock Screen, Keyboard keys, and Menu bar.

### Player State Machine Rule
**Rule**: The UI state MUST strictly follow the hardware `<audio>` state.
- **Enums**: Prefer a `status` enum (`idle`, `loading`, `playing`, `paused`, `error`) over multiple booleans.
- **Reverse Sync**: The UI `isPlaying` state must be updated by the audio element's `onPlay` and `onPause` events, not just the user's click.
- **Error Handling**: If `audio.play()` fails (Autoplay block), the UI must revert to `paused`.

## Internationalization & RTL

### Logical Properties (RTL Rule)
**Rule**: NEVER use physical direction classes (`ml-`, `mr-`, `pl-`, `pr-`, `left-`, `right-`).
**Requirement**: Use Logical Properties to ensure UI correctly mirrors for Right-to-Left (RTL) languages (Arabic, Hebrew).
- `ml-*` -> **`ms-*`** (Margin Start)
- `mr-*` -> **`me-*`** (Margin End)
- `pl-*` -> **`ps-*`** (Padding Start)
- `pr-*` -> **`pe-*`** (Padding End)
- `text-left` -> **`text-start`**
- `border-l` -> **`border-s`**

### Date & Number Formatting
**Rule**: Use `Intl` API or `date-fns` with locale objects for all user-facing dates and numbers. Never manually split date strings.

## File Handling (Standard)
- **Rule**: Do NOT use manual drag/drop event listeners for file ingestion.
- **Library**: **`react-dropzone`**.
- **Implementation**: Use the `useDropzone` hook to manage states (`isDragActive`, `acceptedFiles`).

## Error Handling Tiers
| Tier | Trigger | Action |
|------|---------|--------|
| **User Error** | Operation failure | Toast / Empty State (i18n) |
| **System Error** | Parse failure | `console.warn/error` (silent) |

**Toast i18n rule**: User-facing toasts MUST use `toast.*Key(...)`. Do not surface arbitrary exception strings to users; log details and show a stable i18n key.

## Type-Safe I18n
- **Rule**: All translations must use **`react-i18next`**.
- **Typing**: `t()` function must be type-checked against English keys (see `i18n-guide.md`).

## Testing Standards (The Pyramid)
- **Framework**: Vitest.
- **Store Testing**: Reset store state in `beforeEach`.

## Files & Structure
- **`src/lib`**: Centralize business logic (`storage.ts`, `dateUtils.ts`).
- **`src/components/ui`**: Reserved for shadcn primitives.
- **`src/components/interactive`**: Custom interaction primitives.

## Defensive Engineering

### Layout Robustness (The Long-Text Rule)
**Rule**: NEVER assume text will be short.
- **Titles**: Always use `truncate` or `line-clamp-X`.
- **Containers**: Use `min-w-0` on flex items to ensure they don't push past parent boundaries.
- **Scroll**: Ensure overflow is handled (`overflow-hidden` or `overflow-y-auto`).

### Multi-Tab Singleton
**Rule**: Audio apps must be Singletons per browser.
- **Implementation**: Use a `BroadcastChannel` named `readio_playback`.
- **Action**: When tab A starts playing, send a "PLAY" message. All other tabs (B, C) MUST pause immediately.

### Network Resilience [Implemented]

#### Fallback Chain (Retry Rule)
**Rule**: Keep latency predictable by trying each chain step once, then move on.
- **Target**: One attempt per step (`maxRetries: 0`) in `fetchWithFallback`.
- **Fallback Policy**: On any failure (network/timeout/HTTP), move to the next configured step immediately.
- **Exclusion**: No local backoff retries inside a single step.

#### Circuit Breaker (Metrics-Only)
**Rule**: Track endpoint health without short-circuiting user requests.
- **Implementation**: Failure/success counters are recorded per proxy.
- **Policy**: Circuit "trip" state does not skip requests; all configured steps are still attempted.

### Async Interaction Safety (Race Guard) [Implemented]
**Rule**: UI state must determine the "Last Write". Prevent inconsistencies caused by rapid interactions.

1.  **Request ID Tracking**: Long-running Store actions (like `restoreSession` or processing subtitles) must track a `requestId`. If the ID changes (due to a new user action) before the async operation completes, the stale operation must Abort.
2.  **AbortController**: Search and Fetch operations must use `AbortController` (or TanStack Query's logical equivalent) to cancel standard network requests.
3.  **Playback Lock**: `play()` logic must check if the player status is `loading` or restricted to prevent overlapping audio requests.

### Selection & Lookup UI Transitions (Interaction Safety) [Implemented]
**Rule**: Prevent UI "flicker" and accidental state overlaps during cross-surface transitions.

1.  **Targeted Closure**: Dismissal functions (`closeMenu`, `closeLookup`) MUST verify the `surface` type before execution. A closing "Range Action Menu" must not be able to close an active "Word Context Menu".
2.  **Backdrop Absorption**: High-value UI like `LookupCallout` must use a transparent backdrop to swallow the FIRST click outside (Absorption).
    - **Left-click**: Swallows the event to dismiss the UI safely.
    - **Right-click**: Immediately unmounts to allow the native `contextmenu` event to hit the underlying target (Switch).
3.  **Event Button Guard**: Global pointer listeners for selection finalization should strictly guard for `e.button === 0` (Left Click) to prevent OS right-click gestures from interfering with selection logic.

### Graceful Degradation
- **Rule**: If a local file is missing from DB (e.g., deleted by browser cleanup), show an "File Missing" state instead of crashing.

### Granular Error Boundaries (The Isolation Rule) [Implemented]
**Rule**: UI crashes MUST be isolated to the specific component. A rendering failure in a subtitle or a search result row must NOT crash the entire page.
- **Hierarchy**: Use `ComponentErrorBoundary` for leaf components (`EpisodeRow`, `TranscriptView`, `Sidebar`).
- **Fallback**: The fallback UI must preserve the intended layout height/width to prevent erratic layout shifts when an item fails.
- **Logging**: Errors must be logged via `logError` for diagnosis.
- **Reporting Scope**: Default behavior is **log-only**. Only critical surfaces (e.g., `TranscriptView`) should wire `onError` to `reportError` for telemetry.
- **Recovery**: Include a local "Retry" button to reset the boundary state.

### Async Error & Promise Safety [Implemented]
**Rule**: All asynchronous operations MUST be explicit, handled, and cancellation-aware.

1.  **Explict Handling**: NEVER allow "floating" Promises.
    - All async calls must be `await`ed, or
    - Prefixed with `void` to signal intentional fire-and-forget, with local `.catch(logError)` handling.
2.  **Store Action Contracts**:
    - Actions touching DB/Network must wrap logic in `try...catch`.
    - Surface errors via `status` (loading/error) or `toast.*Key`.
    - If `status` exists, it is the primary UX signal. Duplicate toasts for transient aborts should be avoided.
3.  **Cancellation Guards**:
    - Async store actions MUST accept an optional `AbortSignal`.
    - UI state updates MUST be guarded with `if (signal?.aborted) return`.
    - This prevents "ghost" state updates after a user has navigated away or changed context.
4.  **Lint Enforcement**:
    - The `nursery/noFloatingPromises` [Biome](https://biomejs.dev/) rule is enabled as an `error`.

## Anti-Patterns: File Drop
❌ **Wrong**: Native listeners
```tsx
<div onDrop={e => handleFiles(e.dataTransfer.files)} />
```
✅ **Right**: React Dropzone
```tsx
const { getRootProps } = useDropzone({ onDrop })
<div {...getRootProps()} />
```

## Enforced Architecture Rules

### DB Access Guard (Store-First Persistence) [Implemented]
**Rule**: UI components (`src/components/**`, `src/routeComponents/**`) MUST NOT import ANYTHING from `dexieDb` (including type-only imports).

- **Access Pattern**: Components call **hooks or store actions** only. Hooks/stores call `DB.*`.
- **Type Pattern**: UI imports DB entity types from `lib/db/types` (not from `dexieDb`).
- **Guard Script**: `pnpm lint:db-guard` enforces this rule locally and in CI.

**Example**:
```tsx
// ❌ Wrong - direct dexieDb import
import type { FileTrack } from '../../lib/dexieDb'

// ✅ Right - use type surface module
import type { FileTrack } from '../../lib/db/types'
```

**Rationale**: Enforces clear separation between UI layer and persistence implementation, preventing tight coupling and enabling future persistence refactors without UI churn.

**Verification**:
```bash
# Check for violations
pnpm --filter @readio/lite lint:db-guard

# Should output: ✅ No direct DB access violations found!
```
