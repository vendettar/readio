# Routing Strategy

Readio Lite uses **TanStack Router** for file-based routing.
This document defines the URL structure, navigation patterns, and Slug strategy.

## 1. URL Structure

| Route | Pattern | Description |
|---|---|---|
| Home | `/` | Uses `beforeLoad` to redirect to `/files` if no session found. |
| Podcast | `/$country/podcast/$id` | Country-scoped Podcast Show page. `$country` is the route SSOT for content-region resolution. |
| Episode | `/$country/podcast/$id/episode/$episodeId` | Country-scoped Episode Detail. See "Slug Strategy" below. |
| Files | `/files/*` | Local file browser. Uses wildcard for folder paths. |
| Legal | `/legal/privacy`, `/legal/terms` | Privacy Policy and Terms of Service. |

## 2. Slug Strategy (Strict Slug-Only)

Episode URLs use a **strict slug-only** format. Raw GUIDs are not supported as route params.

**Format**: `{titleSlug}-{shortId8}`
- **titleSlug**: Generated from the episode title (`generateSlug()`), lowercase ASCII, hyphens, fallback `episode`.
- **shortId8**: Fixed 8-character alnum token generated by `toShortId8Token(fullId)`. If source ID is shorter than 8, deterministic hash padding is applied.

**Example**:
`/us/podcast/123/episode/how-i-built-this-a1b2c3d4`

**Utilities** (`lib/slugUtils.ts`):
- `generateSlug(text)` — normalize title to URL-safe slug.
- `generateSlugWithId(title, fullId)` — produce `{titleSlug}-{shortId8}`.
- `parseSlugWithId(value)` — extract `{ titleSlug, shortId }` where `shortId` must be exact 8 chars.

**Resolution Logic** (`useEpisodeResolution`):
1. `parseSlugWithId(episodeId)` extracts `shortId`.
2. Match by strict `shortId8` token equality (`toShortId8Token(candidateId) === shortId`).
3. **Collision handling** (deterministic):
   - Prefer candidate whose `generateSlug(title)` matches `titleSlug`.
   - Otherwise: `pubDate` descending, then `id` lexical ascending.

**Canonical Enforcement** (`PodcastEpisodeDetailPage`):
- After episode resolves, if the route slug differs from the canonical slug (`generateSlugWithId(episode.title, episode.id)`), the URL is replaced via `navigate({ replace: true })`.

**Route Validation** (`routes/$country/podcast/$id/episode/$episodeId.tsx`):
- `episodeId` must pass `parseSlugWithId()` validation in `beforeLoad`.
- Invalid slugs redirect to `/$country/podcast/$id`.

## 3. Navigation Rules

- **Pre-loading**: Use `loader` in route definitions to fetch Podcast Feed data *before* rendering the component.
- **Redirects**: Do NOT use `return <Navigate />` in the render body. Use `throw redirect()` inside `beforeLoad` or `loader`.
- **Preservation**: The Audio Player component lives in `__root.tsx` and persists across all route changes. The persistent `PlayerSurfaceFrame` manages both docked and full reading views without unmounting.
- **Canonical URL Hygiene (All Entrypoints)**: Explore, Search page, Global Search overlay, and Library pages (History/Favorites/Subscriptions) must all navigate to path-only podcast content routes (`/$country/podcast/$id` / `/$country/podcast/$id/episodes` / `/$country/podcast/$id/episode/$episodeId`) without query hints (`source/feedUrl/audioUrl/providerEpisodeId/sessionId`).
- **Legacy Query Hint Handling**: Content routes do not strictly reject legacy query hints; they are tolerated and stripped during search validation (effective search payload becomes `{}`), and route correctness remains path-only.
- **Episode Routes Contract Tightening (Instruction 116)**: `/$country/podcast/$id/episodes` and `/$country/podcast/$id/episode/$episodeId` use route-local `z.object({})` validation and no longer parse shared legacy detail-search schema. Legacy hints are ignored only (not correctness inputs).
- **Legacy Hint Contract (Instruction 117)**: `/$country/podcast/*` keeps tolerant `z.object({})` search schemas intentionally. Hints are ignored/stripped for compatibility and never used as correctness inputs.
- **Legacy Country Resolver Status (Instruction 119)**: `libraryCountryResolver` is fully decommissioned and must not be used for content-route correctness. Country authority remains route param `$country` only.
- **Transition Metadata Contract**: `fromLayoutPrefix` is transition-only metadata and must travel via `location.state` (typed state), never via URL `search`.
- **`location.state` Boundary Rule (Instruction 116f)**: `location.state` is allowed only for transition metadata (e.g., `fromLayoutPrefix`). It must never carry correctness-critical business identity (`country/podcastId/episodeId/feedUrl/...`) because refresh/direct-open/share links do not preserve state.
- **Country Switch Request Lifecycle**: content-route queries are country-keyed and signal-aware, so route-country switches activate new keys and cancel/ignore stale in-flight requests.
