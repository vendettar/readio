# 项目特定领域规则

这些规则专门通过 Readio 业务逻辑。

## 1. 全局搜索架构

- **搜索是一种模式，而非本地过滤器**:
  - 与当前页面内容解耦。
  - 从侧边栏触发。

- **两阶段体验**:
  1. **预览浮层**: 浮动下拉菜单 (限制 3-5 项)。
  2. **结果页面**: 全页路由 `/search?q=query`。

- **分组**:
  - 严格按 **内容类型** 分组 (剧集, 播客)。
  - 状态 (已播放, 已订阅) 是徽章，不是分组。

## 2. 音频与播放

- **Readio 身份**:
  - Readio 是一个**播放音频的阅读工具**，而不仅仅是音乐播放器。

- **逐字稿规则**:
  - **每个单词必须可见**。逐字稿上没有 `text-overflow: ellipsis`。
  - 激活行必须保持居中。
  - **选区与上下文菜单契约**:
    - **分层锚点策略**:
      - **单词与划词选区**: 必须使用稳定的几何锚点（Rect 水平中心，顶部边缘）。这能防止菜单在视口抖动或后续点击中产生漂移。
      - **行内空白处**: 必须使用 pointer-origin 锚点（点击时的 `clientX/Y`），避免菜单跳变到 block 中心。
    - **非模态交互**: 所有选区菜单必须是非模态的 (`modal={false}`)。严禁使用全屏透明阻挡层。必须通过捕获阶段监听器“吸收”第一次域外点击（Outside Click），关闭菜单并恢复焦点，而不触发底层行为（如跳转进度）。
    - **位置与遮挡**: 菜单必须优先使用 `top` placement（出现在点击点上方），以减少对当前正在阅读/合成的字幕文本的遮挡。
    - **视觉一致性**: 所有选区相关表面（单词菜单、划词菜单）必须统一使用垂直式的 `DropdownMenu` 架构与系统主题 (`bg-popover`)。
  - **防回归：原生文本选中拦截**: 为了既支持单词单击词典查询和手写划词（Drag Selection），又防止产生难以消除的原生蓝色文本选中高亮，我们必须在容器级使用配置为 `{ capture: true }` 的 `mousedown` 监听层。在捕获阶段拦截所有 `e.detail >= 2`（双击、三击或更多）的事件并执行最高级 `e.preventDefault()`。手动拖拽（detail 为 1 且伴随位移）产生的选区正常生效。

- **音频播放器生命周期**:
  - `<audio>` 元素在 `__root.tsx` 中是持久的。
  - 必须仔细管理事件监听器以避免内存泄漏。

- **剧集会话完整性**:
  - 如果剧集缺少 `podcastFeedUrl`，不要创建播放会话。

## 3. 文件系统与发现

- **文件密度**:
  - 视图密度设置 (`comfortable` / `compact`) 适用于整个 Files 页面。

- **规范化**:
  - 剧集 ID: 如果可用，使用 GUID；否则回退到 `trackId`。
  - 在路由中放置 `episodeId` 时始终 `encodeURIComponent`。
