# 路由策略

Readio Lite 使用 **TanStack Router** 进行基于文件的路由。
本文档定义了 URL 结构、导航模式和 Slug 策略。

## 1. URL 结构

| 路由 | 模式 | 描述 |
|---|---|---|
| Home | `/` | 使用 `beforeLoad` 重定向到 `/files` (如果未找到会话)。 |
| Podcast | `/$country/podcast/$id` | 带国家前缀的播客详情页。`$country` 是内容地区解析的唯一权威来源。 |
| Episode | `/$country/podcast/$id/episode/$episodeId` | 带国家前缀的单集详情。见下文“Slug 策略”。 |
| Files | `/files/*` | 本地文件浏览器。使用通配符表示文件夹路径。 |
| Legal | `/legal/privacy`, `/legal/terms` | 隐私政策与服务条款。 |

## 2. Slug 策略（严格 Slug-Only）

Episode URL 使用**严格 Slug-Only** 格式。不支持原始 GUID 作为路由参数。

**格式**: `{titleSlug}-{shortId8}`
- **titleSlug**: 由单集标题生成 (`generateSlug()`)，小写 ASCII、连字符，回退为 `episode`。
- **shortId8**: 固定 8 位字母数字 token，由 `toShortId8Token(fullId)` 生成。若源 ID 长度不足 8，则使用稳定哈希补齐。

**示例**:
`/us/podcast/123/episode/how-i-built-this-a1b2c3d4`

**工具函数** (`lib/slugUtils.ts`):
- `generateSlug(text)` — 将标题规范化为 URL 安全 slug。
- `generateSlugWithId(title, fullId)` — 生成 `{titleSlug}-{shortId8}`。
- `parseSlugWithId(value)` — 提取 `{ titleSlug, shortId }`，其中 `shortId` 必须固定 8 位。

**解析逻辑** (`useEpisodeResolution`):
1. `parseSlugWithId(episodeId)` 提取 `shortId`。
2. 通过严格 `shortId8` token 相等匹配（`toShortId8Token(candidateId) === shortId`）。
3. **碰撞处理**（确定性）：
   - 优先选择 `generateSlug(title)` 匹配 `titleSlug` 的候选项。
   - 否则：`pubDate` 降序，然后 `id` 字典序升序。

**规范 Slug 强制** (`PodcastEpisodeDetailPage`):
- 单集解析后，如果路由 slug 与规范 slug (`generateSlugWithId(episode.title, episode.id)`) 不同，则通过 `navigate({ replace: true })` 替换 URL。

**路由验证** (`routes/$country/podcast/$id/episode/$episodeId.tsx`):
- `episodeId` 必须在 `beforeLoad` 中通过 `parseSlugWithId()` 验证。
- 无效 slug 重定向到 `/$country/podcast/$id`。

## 3. 导航规则

- **预加载**: 在路由定义中使用 `loader` 在渲染组件 *之前* 获取 Podcast Feed 数据。
- **重定向**: **不要** 在渲染体中使用 `return <Navigate />`。在 `beforeLoad` 或 `loader` 内部使用 `throw redirect()`。
- **持久化**: Audio Player 组件位于 `__root.tsx` 中，在所有路由更改中持久存在。持久的 `PlayerSurfaceFrame` 管理着 docked 和 full 两种阅读模式，不会被卸载。
- **全入口 URL 规范（Canonical）**：Explore、Search 页、全局搜索 Overlay、以及 Library 页面（History/Favorites/Subscriptions）跳转到播客内容路由时，必须统一为纯路径形态（`/$country/podcast/$id` / `/$country/podcast/$id/episodes` / `/$country/podcast/$id/episode/$episodeId`），不得依赖 `source/feedUrl/audioUrl/providerEpisodeId/sessionId` 查询参数。
- **旧查询提示处理策略**：内容路由不会对旧 query hints 做严格拒绝；会在 search 校验阶段容忍并剥离（最终等效为 `{}`），路由正确性仍仅依赖路径参数。
- **单集路由契约收口（Instruction 116）**：`/$country/podcast/$id/episodes` 与 `/$country/podcast/$id/episode/$episodeId` 已改为路由内 `z.object({})` 校验，不再解析共享的 legacy detail-search schema。旧 hints 仅被忽略，不参与正确性。
- **旧提示契约（Instruction 117）**：`/$country/podcast/*` 保留 `z.object({})` 容忍策略是有意设计。旧 hints 仅被忽略/剥离，不参与正确性判断。
- **旧国家解析器状态（Instruction 119）**：`libraryCountryResolver` 已完全下线，禁止再用于内容路由正确性。国家唯一权威仍是路由参数 `$country`。
- **转场元数据契约**：`fromLayoutPrefix` 仅属于转场元数据，必须通过 `location.state`（类型化 state）传递，禁止通过 URL `search` 读写。
- **`location.state` 边界规则（Instruction 116f）**：`location.state` 只允许承载转场元数据（如 `fromLayoutPrefix`），禁止承载正确性关键业务身份（`country/podcastId/episodeId/feedUrl/...`）。原因是刷新、直开、分享链接都不会保留 state。
- **国家切换请求生命周期**：内容路由查询以 country 参与 key，并透传 `signal`，因此切换 `/$country` 时会启用新 key，同时取消/忽略旧的 in-flight 请求，防止 stale 覆写。
