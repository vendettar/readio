## 运动与空间隐喻

- **共享元素转换 (Active)**: 当持久元素 (如播客封面) 从一个视图移动到另一个视图 (例如 Mini Player 到 Full Player) 时，必须使用共享布局转换 (Framer Motion 中的 `layoutId`) 以保持空间方向。
- **layoutId 策略**：系统使用 `artwork-${id}-player`。为了实现“从单集列表点击后飞向播放器”的视觉效果，列表项、MiniPlayer 和 FullPlayer 必须共享同一个带有 `-player` 后缀的 ID。
- **重复列表处理 (Scoping)**：在包含重复类别的视图中（如 Explore 页面），如果同一节目出现在不同模块，直接共用 ID 会导致 `layoutId` 冲突（图片消失或无响应）。
  - **解决方案**：在布局 ID 前加上模块标识符（例如 `artwork-podcast-topShows-${id}`）。
  - **握手协议**：导航跳转时必须通过类型化 `location.state` 传递 `fromLayoutPrefix`，以便目标详情页匹配对应的转换 ID。该元数据不属于 URL 查询参数契约。
- **视口兼容性**：在 `FullPlayer` 内部，通过 `isDesktop` 条件确保 `layoutId` 仅在当前可见的图片节点上生效，防止 Framer Motion 布局计算冲突导致动画失效。
- **入场动画**: 列表 (文件, 剧集) 不应立即出现。使用 `AnimatedList` bit 实现交错淡入 + 上滑效果。
- **微交互**: 使用 `CountUp` bit 处理数值变化（如存储占用量），提供高级、动态的交互感。
- **应用启动**: 不要显示加载了一半的 UI。在 store 从 Dexie 恢复时，使用带有呼吸动画的居中应用 Logo。使用 `AnimatePresence` 淡出加载器。

## 逐字稿与选区 (Transcript & Selection)

### 锚点策略
系统采用分层锚点策略以平衡稳定性与响应性：
- **几何锚点 (Geometric)**：适用于单词右键菜单和划词选区。UI 锚定在选区的水平中心和顶部边缘。这确保了菜单在视口变化时能始终“吸附”在语义内容上，而不是鼠标点击的位置。
- **指针锚点 (Pointer-Origin)**：适用于在字幕行空白处右键。菜单直接锚定在鼠标点击坐标，以避免跳变到字幕块中心，符合标准操作系统的右键行为。

### 非模态契约
选区与查词组件遵循严格的非模态 (Non-modal) 交互模型：
- **无遮罩 (No Backdrop)**：严禁使用全屏透明阻挡层。
- **交互吸收 (Absorption)**：使用捕获阶段监听器捕获 UI 外部的第一次点击。该点击会关闭当前 UI 并恢复焦点，但不会触发底层的背景行为（如点击导致的进度跳转）。

### 遮挡优化
为了确保播放期间的可读性，所有选区 UI 优先采用 `top` placement。通过出现在焦点上方，最大限度地减少对当前正在朗读或合成的字幕文本的视觉遮挡。

## 列表行交互
  
- **悬停层 (Hover Layer)**：
  - **常规列表**：使用 `bg-accent/50` 作为标准列表行和卡片的背景高亮。
  - **搜索面板 (Search Palette)**：使用实心的 `bg-primary` 背景高亮，以提供最大对比度和清晰的对焦状态 (Decision D081)。
- **微弱分隔 (Subtle Separation)**：使用 `border-border/50` 作为分隔线。
- **智能分隔线 (Smart Dividers)**：分隔线应在活动（悬停）行上自动隐藏或被覆盖：
  - **常规列表**：使用 `:has()` 选择器隐藏相邻分隔线，营造“浮动”对焦卡片的效果。
  - **搜索面板（合并选择）**：在活动条目上使用负上边距 (`-mt-px`) 和 `z-10`，在视觉上覆盖前一项目的分隔线。这避免了 `:has()` 的复杂性，同时确保了锐利、统一的选中矩形。
- **操作可见性**：交互按钮（播放、收藏、菜单）通常应在行悬停时出现或增加不透明度。

- **覆盖内容**: 显示显著的“放开以上传文件”消息，并配以动画图标（如跳动的上传图标）以确认活动状态。

## 搜索与命令面板 (Command Palette)

- **全局入口**：侧边栏的搜索触发器必须采用内嵌集成式的搜索输入框，并在展开时显示高级浮层。
- **键盘优先**：面板内的导航必须支持方向键切换与回车选中（基于 `cmdk`）。
- **快捷键**：必须全局绑定 `Cmd+K` (或 `Ctrl+K`) 以开启搜索面板。
- **视觉美学**：面板必须遵循高级玻璃拟态（`backdrop-blur-xl`, `bg-popover/95`, `backdrop-saturate-150`）。
- **标准徽章系统**：所有状态标识（Subscribed, Favorited, History）必须使用统一的右对齐**图标**样式（text-primary 颜色）。当多状态共存时（如收藏且已听过），必须并列显示所有图标，不得相互覆盖。
- **智能高亮 (Smart Highlighting)**：当用户键入时，搜索建议必须在视觉上区分匹配字符（亮前景色）和其余文本（静默色），以提供为什么建议该结果的清晰反馈。
- **选择清理 (Selection Cleanup)**：结果面板和搜索框之间的鼠标移动必须触发选择重置 (`onMouseEnter` / `onMouseLeave`)，以维持干净的交互流，避免“粘滞”的选中状态。

## 快捷键与手势 (Hotkeys & Gestures)

- **全局快捷键**:
  - `Space`: 切换播放/暂停（全局有效，输入框内输入时除外）。
  - `Arrow Left/Right`: 后退/前进 15 秒。
  - `Cmd/Ctrl + K`: 聚焦搜索。
  - `Esc`: 关闭遮罩层或退出全屏播放器。
- **触摸交互**:
  - **下拉关闭 (Swipe Down)**: 在全屏播放器容器（尤其是顶部拖拽柄区域）向下滑动必须能折叠播放器并返回主视图。
  - **拖拽柄 (Drag Handle)**: 全屏播放器顶部应显示一个微妙的视觉柄（药丸型、低不透明度），以提示下拉行为。
