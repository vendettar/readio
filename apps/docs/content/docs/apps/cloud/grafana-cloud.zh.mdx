# Grafana Cloud 集成

Readio Cloud 使用零 Agent 的应用侧 push 路径处理后端指标和日志，并使用 Grafana Faro 处理前端 RUM。部署不会安装 Grafana Alloy、Promtail、node_exporter、Docker log driver 或系统日志 agent。

## 1. 后端指标 (OTLP Push)

后端服务通过 OTLP HTTP 协议直接向 Grafana Cloud 推送指标。**无需在本机安装 Alloy/Agent。**

### 获取凭证步骤：
1. 登录 [Grafana Cloud](https://grafana.com/products/cloud/) 控制台。
2. 在左侧导航栏点击 **Connections** -> **Add new connection**。
3. 搜索并选择 **OpenTelemetry**。
4. 在配置页面中找到以下信息：
   - **Endpoint**: 类似 `https://otlp-gateway-prod-xxx.grafana.net/otlp`。
   - **Username / Instance ID**: 一个数字 ID。
   - **Password / API Token**: 点击 "Generate now" 生成一个 Token（需要 `MetricsPublisher` 权限）。

### 需要配置到 GitHub 的变量：
- Variable: `READIO_GRAFANA_OTLP_ENDPOINT`
- Variable: `READIO_GRAFANA_OTLP_INSTANCE_ID`
- Secret: `READIO_GRAFANA_OTLP_TOKEN`


---

## 2. 前端 RUM (Grafana Faro)

前端监控直接从用户浏览器向 Grafana Faro Collector 上报数据。

### 获取凭证步骤：
1. 在左侧导航栏点击 **Frontend** (或搜索 Faro)。
2. 点击 **Create new app**。
3. App Name 填 `readio-cloud`。
4. 创建后，在 "Instrumentation" 标签页找到 **URL** (Collector URL)。

### 需要配置到 GitHub Secrets 的变量：
- `VITE_GRAFANA_FARO_URL`: 类似 `https://faro-collector-prod-xxx.grafana.net/collect/xxx`
- `VITE_GRAFANA_FARO_APP_NAME`: `readio-cloud`
- `VITE_GRAFANA_FARO_ENV`: `production` 或 `preproduction`

---

## 3. 后端日志 (Loki Push)

后端可以把脱敏后的应用日志直接推送到 Grafana Cloud Logs/Loki。这条 pipeline 与 OTLP 指标独立，并且只有三个必需 Loki 值全部配置后才会启用。

### 需要配置到 GitHub 的变量：
- Variable: `READIO_GRAFANA_LOKI_URL`：Loki push endpoint，通常以 `/loki/api/v1/push` 结尾。
- Variable: `READIO_GRAFANA_LOKI_USER`：Grafana Cloud Logs user / instance ID。
- Secret: `READIO_GRAFANA_LOKI_TOKEN`：Grafana Cloud Logs token。

可选的仅后端变量是 `READIO_LOKI_LOG_LEVEL`、`READIO_LOKI_BATCH_SIZE`、`READIO_LOKI_FLUSH_INTERVAL_SECONDS` 和 `READIO_LOKI_QUEUE_SIZE`。

Loki labels 只能是 `service`、`env` 和 `level`。Request ID、route、upstream host、用户输入、URL、headers、cookies、tokens、本地路径和原始错误不得作为 labels。

## 4. Grafana 日志查询

Grafana Explore 查询：

```logql
{service="readio-cloud", env="preproduction"}
```

```logql
{service="readio-cloud", env="preproduction", level="error"}
```

```logql
{service="readio-cloud", env="production"}
```

```logql
{service="readio-cloud", env="production", level="error"}
```

日志面板查询：

```logql
{service="readio-cloud", env="$env"} | json
```

持续错误告警查询：

```logql
sum(count_over_time({service="readio-cloud", env="production", level="error"}[5m])) > 0
```

告警阈值应在掌握生产基线流量和预期错误量后再调优。

## 5. 线上验收清单

真实 Grafana Cloud 验证需要部署凭证和 Grafana 访问权限。如果实现阶段无法访问这些资源，运维方应在生产最终验收前完成：

1. 使用 Loki 环境变量部署到 preproduction。
2. 通过正常 Cloud API 流量生成一条 info 日志和一条 warning/error 日志。
3. 在 Grafana Explore 查询 `{service="readio-cloud", env="preproduction"}`。
4. 查询 `{service="readio-cloud", env="preproduction", level="error"}`。
5. 确认日志在配置的 flush interval 内到达。
6. 确认脱敏日志行不包含 secrets、完整 Referer query、本地文件路径、transcript text 或 authorization headers。
7. 确认启用日志投递后，00B 指标仍正常到达。

## 6. 验证数据流入
- **指标**: 在 Grafana Explore 中搜索 `readio_cloud_http_request_duration_seconds`。
- **日志**: 在 Grafana Explore 中使用上面的 Loki 查询。
- **RUM**: 在 Grafana Frontend 页面查看用户会话数据。
