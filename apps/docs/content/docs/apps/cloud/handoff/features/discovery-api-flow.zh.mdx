## 1. 第一跳 API

这一章只记录用户入口页面直接触发的外部 discovery API。

### 1.1 Apple Search Podcasts

- 页面/入口：
  - 搜索页的节目结果
- 上游：
  - Apple Search API
- 常见输入：
  - `term`
  - `country`
  - `limit`
- Readio 实际使用的核心输出：
  - `podcastItunesId`
  - `title`
  - `author`
  - `artwork`
  - 当前 schema 里仍保留的搜索摘要字段
- 下一步：
  - 当需要 canonical podcast metadata 或节目详情富化时，继续走 Podcast Index `podcasts/byitunesid`

### 1.2 Apple Search Episodes

- 页面/入口：
  - 搜索页的单集结果
- 上游：
  - Apple Search API
- 常见输入：
  - `term`
  - `country`
  - `limit`
- Readio 实际使用的核心输出：
  - `podcastItunesId`
  - `title`
  - `showTitle`
  - `artwork`
  - `episodeUrl`
  - `guid`
  - `releaseDate`
  - `trackTimeMillis`
  - `shortDescription`
- 下一步：
  - 当需要 canonical podcast metadata 时，继续走 Podcast Index `podcasts/byitunesid`
  - 当需要页面渲染用单集数据时，读取 Cloud SQLite retained snapshot。snapshot 已存在后，分页列表读取是 DB-only；冷列表 miss 可以先 bootstrap 有界 snapshot。直接单集详情是 SQLite 直接查找，不负责 cold 或 stale 刷新。

### 1.3 Apple Top Podcasts

- 页面/入口：
  - Explore Top Shows
- 上游：
  - Apple top podcasts chart feed
- 常见输入：
  - `country`
  - `limit`
- Readio 实际使用的核心输出：
  - `podcastItunesId`
  - `title`
  - `author`
  - `artwork`
  - `genres`
- 下一步：
  - 当用户进入节目页或需要 canonical podcast metadata 时，继续走 Podcast Index `podcasts/byitunesid`

### 1.4 Apple Top Episodes

- 页面/入口：
  - Explore Top Episodes
- 上游：
  - Apple top episodes chart feed
- 常见输入：
  - `country`
  - `limit`
- Readio 实际使用的核心输出：
  - `podcastItunesId`
  - `title`
  - `author`
  - `artwork`
  - `genres`
- 下一步：
  - 这个第一跳只负责提供 Explore Top Episodes 板块展示所需的数据
  - 用户点击后，后续解析从 show-level context 继续，起点是 Podcast Index `podcasts/byitunesid`

### 1.5 Podcast Index Podcasts Batch By GUID

- 页面/入口：
  - Explore Editor's Picks
- 上游：
  - Podcast Index `podcasts/batch/byguid`
- 常见输入：
  - 由 editor-pick surface 持有的一组 podcast GUID
- Readio 实际使用的核心输出：
  - editor-pick surface 使用的收窄节目 DTO
- 下一步：
  - 当用户打开某个 picked show 时，继续走节目页 lookup，也就是 Podcast Index `podcasts/byitunesid`
  - 当该节目页后续需要单集数据时，继续走 Cloud SQLite retained snapshot 路由

## 2. 第二跳与后续解析

这一章记录第一跳之后、或者已有 route / cache / canonical podcast context 时才会触发的 API。

### 2.1 Podcast Index Podcast By iTunes ID

- 上游：
  - Podcast Index `podcasts/byitunesid`
- 用途：
  - 获取 canonical podcast metadata
  - 节目页 lookup
  - 富化 Apple 第一跳结果
- 上一跳：
  - Apple Search Podcasts
  - Apple Search Episodes
  - Apple Top Podcasts
  - Apple Top Episodes
  - 或者基于 `podcastItunesId` 直接进入节目页
- 依赖上一跳提供：
  - `podcastItunesId`
- 向下游产出：
  - canonical podcast contract

### 2.2 Cloud SQLite 单集 Snapshot

- 路由：
  - Cloud `GET /api/v1/discovery/podcasts/:itunesId/episodes`
- 刷新来源：
  - Podcast Index `episodes/byitunesid`，通过 cold bootstrap 和节目详情刷新路径使用
- 用途：
  - 从 retained snapshot 提供节目页单集预览
  - 全部单集页基于 DB-only retained snapshot 分页
  - 直接进入的单集详情 hydration
  - 页面渲染需要的 canonical 字段，例如 transcript、chapters、season/episode number、纯文本 description
- 上一跳：
  - Podcast Index `podcasts/byitunesid`
- 依赖上一跳提供：
  - `podcastItunesId`
- 向下游产出：
  - 以稳定单集身份为键的 normalized retained episode rows
  - 通过 `podcastItunesId + episodeGuid` 直接返回单集详情
  - 行对外暴露 `guid` 和必填 `artworkUrl`；后端无法归一化出合法 artwork URL 的行会被丢弃
- 列表模式规则：
  - `GET /episodes?limit=&offset=` 是 DB-only，用户翻页时不得调用 PodcastIndex
- 详情模式规则：
  - `GET /episodes?episodeGuid=...` 只按 `podcastItunesId + episodeGuid` 做 SQLite 直接查找，不触发 PodcastIndex 新鲜度检查

### 2.3 已退役的 Discovery Feed Route

- 状态：
  - `GET /api/v1/discovery/feed` 已退役，不得再被描述为活跃的 Cloud API
- 当前契约：
  - repo-owned 的 Cloud 运行时契约已不再暴露或依赖旧的 feed transport 字段
  - 活跃的 Cloud 页面渲染与 discovery routing 不会再按旧的 feed transport identity 拉取 RSS
- 影响：
  - Cloud 当前不存在用于节目页 / 单集列表 / 单集详情渲染的 RSS feed XML 二跳 API

### 2.4 单集解析顺序

- 第一步：
  - 从 route 或 compact key 解析出目标稳定单集身份
- 第二步：
  - 通过 Podcast Index `podcasts/byitunesid` 获取 canonical podcast metadata
- 第三步：
  - 向 Cloud SQLite snapshot 路由请求基于 `podcastItunesId + episodeGuid` 的直接单集详情
  - 单集详情不会刷新 snapshot；目标不在 retained window 中时直接进入确定性的 `episodeNotFound`
- 第四步：
  - 如果目标 `guid` 不在 retained snapshot 中，直接进入确定性的 `episodeNotFound`

## 3. 页面总览

| 页面/入口 | 第一跳 | 第二跳 | 后续解析 |
|---|---|---|---|
| 搜索节目 | Apple Search Podcasts | 需要时走 Podcast Index `podcasts/byitunesid` | 用户进入内容页后走 Cloud SQLite retained snapshot |
| 搜索单集 | Apple Search Episodes | 需要时走 Podcast Index `podcasts/byitunesid` | 单集解析走 Cloud 直接单集详情 lookup |
| Explore Top Shows | Apple Top Podcasts | 用户进入节目页后走 Podcast Index `podcasts/byitunesid` | 在节目/单集详情流中走 Cloud SQLite retained snapshot |
| Explore Top Episodes | Apple Top Episodes | 用户点击后，从 Podcast Index `podcasts/byitunesid` 的 show context 继续 | 在后续详情/解析链路里走 Cloud SQLite retained snapshot |
| Explore Editor's Picks | Podcast Index `podcasts/batch/byguid` | 用户进入节目页后走 Podcast Index `podcasts/byitunesid` | 在节目/单集详情流中走 Cloud SQLite retained snapshot |
| 节目页 | 无 | Podcast Index `podcasts/byitunesid` | Cloud SQLite retained snapshot |
| 单集详情页 | 无 | Podcast Index `podcasts/byitunesid` | Cloud 直接单集详情 lookup |

## 4. 明确不使用

- Readio 当前 active discovery flow 不使用 Podcast Index `episodes/byguid`
- Readio 当前仍然使用 Podcast Index `podcasts/batch/byguid` 作为 Editor's Picks 的第一跳
- Readio 不再把 `/api/v1/discovery/feed` 视为活跃 discovery route，而且 repo-owned 的 Cloud 运行时契约也不再暴露旧的 feed transport 字段
