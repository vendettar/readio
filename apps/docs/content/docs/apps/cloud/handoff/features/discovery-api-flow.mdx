## 1. First Hop APIs

These are the first external discovery APIs used directly by user-facing entry surfaces.

### 1.1 Apple Search Podcasts

- Surface:
  - Search podcast results
- Upstream:
  - Apple Search API
- Typical input:
  - `term`
  - `country`
  - `limit`
- Core output used by readio:
  - `podcastItunesId`
  - `title`
  - `author`
  - `artwork`
  - other search-specific summary fields still kept in the active schema
- Next step:
  - When canonical podcast metadata or show-detail enrichment is needed, continue to Podcast Index `podcasts/byitunesid`

### 1.2 Apple Search Episodes

- Surface:
  - Search episode results
- Upstream:
  - Apple Search API
- Typical input:
  - `term`
  - `country`
  - `limit`
- Core output used by readio:
  - `podcastItunesId`
  - `title`
  - `showTitle`
  - `artwork`
  - `episodeUrl`
  - `guid`
  - `releaseDate`
  - `trackTimeMillis`
  - `shortDescription`
- Next step:
  - When canonical podcast metadata is needed, continue to Podcast Index `podcasts/byitunesid`
  - When page-rendering episode data is needed, read the Cloud SQLite retained snapshot. Paginated list reads are DB-only after a snapshot exists; a cold list miss may bootstrap the bounded snapshot. Direct episode detail is SQLite direct lookup and does not own cold or stale refresh.

### 1.3 Apple Top Podcasts

- Surface:
  - Explore Top Shows
- Upstream:
  - Apple top podcasts chart feed
- Typical input:
  - `country`
  - `limit`
- Core output used by readio:
  - `podcastItunesId`
  - `title`
  - `author`
  - `artwork`
  - `genres`
- Next step:
  - When the user enters the show or canonical podcast metadata is needed, continue to Podcast Index `podcasts/byitunesid`

### 1.4 Apple Top Episodes

- Surface:
  - Explore Top Episodes
- Upstream:
  - Apple top episodes chart feed
- Typical input:
  - `country`
  - `limit`
- Core output used by readio:
  - `podcastItunesId`
  - `title`
  - `author`
  - `artwork`
  - `genres`
- Next step:
  - This first hop only provides the Explore Top Episodes surface payload
  - After click-through, downstream resolution continues from show-level context, starting with Podcast Index `podcasts/byitunesid`

### 1.5 Podcast Index Podcasts Batch By GUID

- Surface:
  - Explore Editor's Picks
- Upstream:
  - Podcast Index `podcasts/batch/byguid`
- Typical input:
  - array of podcast GUIDs owned by the editor-pick surface
- Core output used by readio:
  - editor-pick podcast rows in the narrowed Readio contract
- Next step:
  - When the user opens a picked show, continue to show-page lookup using Podcast Index `podcasts/byitunesid`
  - When the opened show later needs episode data, continue to the Cloud SQLite retained snapshot route

## 2. Second Hop And Later Resolution

These APIs are used after a first-hop result, route, or previously cached data already exists.

### 2.1 Podcast Index Podcast By iTunes ID

- Upstream:
  - Podcast Index `podcasts/byitunesid`
- Used for:
  - canonical podcast metadata
  - show page lookup
  - enriching first-hop Apple results
- Previous hop:
  - Apple Search Podcasts
  - Apple Search Episodes
  - Apple Top Podcasts
  - Apple Top Episodes
  - route entry by `podcastItunesId`
- Depends on previous hop providing:
  - `podcastItunesId`
- Produces:
  - canonical podcast contract used by show/detail flows

### 2.2 Cloud SQLite Episode Snapshot

- Route:
  - Cloud `GET /api/v1/discovery/podcasts/:itunesId/episodes`
- Refresh source:
  - Podcast Index `episodes/byitunesid` through cold bootstrap and podcast detail refresh paths
- Used for:
  - show page episode preview from the retained snapshot
  - episodes page DB-only pagination over the retained snapshot
  - direct-entry episode detail hydration
  - canonical page-rendering fields such as transcript, chapters, season/episode numbers, and plain-text description
- Previous hop:
  - Podcast Index `podcasts/byitunesid`
- Depends on previous hop providing:
  - `podcastItunesId`
- Produces:
  - normalized retained episode rows keyed by stable episode identity
  - direct episode detail by `podcastItunesId + episodeGuid`
  - rows expose `guid` and required `artworkUrl`; rows without a valid backend-normalized artwork URL are dropped
- List-mode rule:
  - `GET /episodes?limit=&offset=` is DB-only and must not call PodcastIndex when users flip pages
- Detail-mode rule:
  - `GET /episodes?episodeGuid=...` is a direct SQLite lookup by `podcastItunesId + episodeGuid` and must not trigger PodcastIndex freshness checks

### 2.3 Retired Discovery Feed Route

- Status:
  - `GET /api/v1/discovery/feed` is retired and must not be described as an active Cloud API
- Active contract:
  - repo-owned Cloud runtime contracts no longer expose or depend on the legacy feed transport field
  - active Cloud page rendering and discovery routing do not fetch RSS by legacy feed transport identity
- Implication:
  - there is no active second-hop Cloud discovery API that parses RSS feed XML for show / episodes / detail rendering

### 2.4 Episode Resolution Order

- Step 1:
  - resolve route or compact key into a target stable episode identity
- Step 2:
  - resolve canonical podcast metadata through Podcast Index `podcasts/byitunesid`
- Step 3:
  - ask the Cloud SQLite snapshot route for direct episode detail by `podcastItunesId + episodeGuid`
  - episode detail does not refresh the snapshot; absence from the retained window resolves as deterministic `episodeNotFound`
- Step 4:
  - if the target `guid` is absent from the retained snapshot, end in deterministic `episodeNotFound`

## 3. Surface Overview

| Surface | First Hop | Second Hop | Later Resolution |
|---|---|---|---|
| Search podcasts | Apple Search Podcasts | Podcast Index `podcasts/byitunesid` when needed | Cloud SQLite retained snapshot when the user enters page-rendering routes |
| Search episodes | Apple Search Episodes | Podcast Index `podcasts/byitunesid` when needed | Direct Cloud episode detail lookup for episode resolution |
| Explore Top Shows | Apple Top Podcasts | Podcast Index `podcasts/byitunesid` after user enters a show | Cloud SQLite retained snapshot inside show/detail flow |
| Explore Top Episodes | Apple Top Episodes | After click-through, continue from Podcast Index `podcasts/byitunesid` show context | Cloud SQLite retained snapshot inside later detail/resolution flow |
| Explore Editor's Picks | Podcast Index `podcasts/batch/byguid` | Podcast Index `podcasts/byitunesid` after show entry | Cloud SQLite retained snapshot inside show/detail flow |
| Show page | none | Podcast Index `podcasts/byitunesid` | Cloud SQLite retained snapshot |
| Episode detail page | none | Podcast Index `podcasts/byitunesid` | Direct Cloud episode detail lookup |

## 4. Explicit Non-Usage

- Readio does not currently use Podcast Index `episodes/byguid` in the active discovery flow.
- Readio still uses Podcast Index `podcasts/batch/byguid` for Editor's Picks.
- Readio no longer treats `/api/v1/discovery/feed` as an active discovery route, and repo-owned Cloud runtime contracts no longer expose the legacy feed transport field.
