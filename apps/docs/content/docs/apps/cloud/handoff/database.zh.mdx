---
title: 数据库与持久化
---

## 1. 持久化模型

Cloud 前端仍然通过 IndexedDB 和 localStorage 做浏览器本地持久化。

- IndexedDB：元数据、sessions、downloads、subtitles、browser-local library state
- localStorage：仅非敏感轻量设置
- `credentials` 表：按需要保存浏览器本地 provider credential

## 2. 关键约束

Cloud 后端当前并没有替代浏览器本地图书馆模型。

- Favorites、Subscriptions、History、下载音频、本地 transcript 产物仍然只存在于浏览器
- 清除站点数据仍会删除它们

## 3. Runtime Cache 现状

- Cloud 中 discovery / search / feed 前端已不再依赖长期持久化的 IndexedDB 缓存
- 现在 discovery 缓存主要依赖：
  - 前端 TanStack Query 内存缓存
  - 后端 Apple / PodcastIndex relay 的内存缓存
- `runtime_cache` 仍存在，但只服务于当前仍有 owner 的浏览器运行时缓存

## 4. 对 Cloud UI 仍然重要的表

- `sessions`
- `audioBlobs`
- `subtitles`
- `remote_transcripts`
- `folders`
- `tracks`
- `local_subtitles`
- `subscriptions`
- `favorites`
- `settings`
- `credentials`
- `runtime_cache`

## 5. 运行规则

Cloud 前端文档必须如实描述 browser-local persistence：

- 没有账号同步
- 没有服务端恢复
- 不保证长期归档
