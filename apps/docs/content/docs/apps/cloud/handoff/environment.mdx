## 1. Runtime Split

- `apps/cloud-ui` is built as a static frontend bundle.
- `apps/cloud-api` serves the bundle and owns same-origin API routes.

## 2. Frontend Runtime Config

Cloud frontend runtime config is emitted by the backend through `/env.js`.

- Browser-public values must stay on the explicit allowlist.
- Real upstream secrets must not be exposed through browser runtime config.
- `apps/cloud-ui/public/env.js` is only a dev/default template.

## 3. Discovery/Media Contract

- Discovery/search/feed requests are same-origin through Cloud backend routes.
- Browser media remains direct-first where the current Cloud contract allows it.
- Backend fallback/proxy behavior is documented separately in handoff audit docs.

## 4. PWA Status

Cloud currently keeps Workbox/PWA disabled unless a later instruction explicitly restores it.

## 5. Prometheus Metrics Contract

`apps/cloud-api` exposes `/metrics` for host-side Prometheus scraping.

- `READIO_METRICS_TOKEN` is backend-only runtime config; keep it out of `/env.js` and browser allowlists.
- Empty or unset `READIO_METRICS_TOKEN` disables `/metrics` with `404`; configured tokens require `Authorization: Bearer <token>`.
- Application metrics are additive to Go/process collectors:
  - `readio_cloud_http_request_duration_seconds`
  - `readio_cloud_upstream_request_duration_seconds`
  - `readio_cloud_upstream_errors_total`
  - `readio_cloud_asr_relay_requests_total`
- Metric labels use closed low-cardinality enums for `route`, `provider`, `status_class`, `error_class`, `cache_status`, and `mode`.
- Metric labels must not include full URLs, query strings, route IDs, search terms, GUIDs, header names/values, raw errors, request/response bodies, audio bytes, or transcript text.

## 6. Loki Log Shipping Contract

Cloud API logs remain stdout/systemd-journal logs produced by Go `slog`.

- Host-side Grafana Alloy reads the `readio-cloud.service` journal and ships logs to Grafana Cloud Loki.
- Do not add an app-side Go Loki client.
- Grafana Cloud Loki endpoint, user, and token are Alloy/server-owned secrets; keep them out of `/env.js`, frontend `.env`, Vite config, and static frontend assets.
- Recommended Loki labels are low-cardinality only: `job=readio-cloud`, `env=production|preproduction`, and `service=readio-cloud-api`.
- Loki labels must not include full URLs, route params, user input, raw errors, request IDs, headers, cookies, tokens, audio bytes, transcript text, or local paths.

## 7. Faro Browser RUM Contract

Cloud UI Faro telemetry is optional and browser-public-config driven.

- `VITE_GRAFANA_FARO_URL` empty disables Faro.
- `VITE_GRAFANA_FARO_SAMPLE_RATE` defaults to `0`; invalid or out-of-range values are safe.
- Faro collector URL, app name, env, and sample rate are public metadata, not credentials.
- No Grafana write credential, admin token, metrics token, relay secret, provider credential, Basic Auth value, local media, transcript text, or `/ops` session state may be read or exported by Faro.
- Sanitized schema diagnostics may include only schema name, provider enum, route class, issue path, issue code, and coarse error class.
- Faro initialization must fail open and never block app boot.

## 8. Grafana Cloud Host Monitoring Contract

Grafana Cloud observability is additive to `/ops` and stays outside app-side remote write paths.

- Host-side Grafana Alloy owns three pipelines: systemd journal to Loki, local `/metrics` scrape to Grafana Cloud metrics, and host metrics to Grafana Cloud metrics.
- `/metrics` should be scraped from localhost or an internal-only path and remains protected by `READIO_METRICS_TOKEN`.
- `/metrics`, `/admin/*`, and `/ops` should not be publicly exposed without outer restrictions such as Cloudflare, nginx, VPN, or IP allowlists.
- Required host monitoring covers CPU, memory, disk usage, disk free under `/opt/readio/shared/data`, SQLite WAL growth, and transcript asset storage pressure under `/opt/readio/shared/data/podcast/transcripts`.
- Expected dashboards cover upstream latency by provider/route, upstream errors by provider/error class, ASR relay request/error rates, host disk pressure, Loki error discovery for discovery/proxy/ASR, and Faro crash/schema diagnostic rates.
- Suggested alerts cover shared-data disk free, sustained `5xx` or non-`none` error-class rates, ASR relay failure spikes, PodcastIndex/Apple p95 latency spikes, and Faro crash spikes.
- Cost controls require deliberate log retention, conservative Faro sampling, low-cardinality labels, and no raw log fields promoted to labels.
- Rollback disables Alloy pipelines and clears Faro public config while leaving `/ops` and `/admin/*` intact.
