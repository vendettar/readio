## 1. Runtime Split

- `apps/cloud-ui` is built as a static frontend bundle.
- `apps/cloud-api` serves the bundle and owns same-origin API routes.

## 2. Frontend Runtime Config

Cloud frontend runtime config is emitted by the backend through `/env.js`.

- Browser-public values must stay on the explicit allowlist.
- Real upstream secrets must not be exposed through browser runtime config.
- `apps/cloud-ui/public/env.js` is only a dev/default template.

## 3. Discovery/Media Contract

- Discovery/search/feed requests are same-origin through Cloud backend routes.
- Browser media remains direct-first where the current Cloud contract allows it.
- Backend fallback/proxy behavior is documented separately in handoff audit docs.

## 4. PWA Status

Cloud currently keeps Workbox/PWA disabled unless a later instruction explicitly restores it.

## 5. Application Metrics Contract

`apps/cloud-api` pushes application and app-owned host-critical metrics directly to Grafana Cloud through OTLP when the Grafana environment variables are configured.

- Required OTLP runtime config is `READIO_GRAFANA_OTLP_ENDPOINT`, `READIO_GRAFANA_OTLP_INSTANCE_ID`, and `READIO_GRAFANA_OTLP_TOKEN`.
- Discovery CORS is owned by `READIO_DISCOVERY_ALLOWED_ORIGINS`; proxy CORS remains owned by `READIO_PROXY_ALLOWED_ORIGINS`.
- Browser-public English dictionary config is deployed as `READIO_EN_DICTIONARY_API_URL` and `READIO_EN_DICTIONARY_API_TRANSPORT` and emitted only through backend-generated `/env.js`.
- `READIO_GRAFANA_OTLP_TOKEN` is backend-only runtime config; keep it out of `/env.js` and browser allowlists.
- The current 00A deployment does not expose a Prometheus scrape endpoint and does not install local Prometheus or Grafana Alloy.
- Application metrics currently include:
  - `readio_cloud_http_request_duration_seconds`
  - `readio_cloud_upstream_request_duration_seconds`
  - `readio_cloud_upstream_errors_total`
  - `readio_cloud_asr_relay_requests_total`
- App-owned host metrics currently include:
  - `readio_cloud_process_goroutines`
  - `readio_cloud_process_heap_alloc_bytes`
  - `readio_cloud_process_memory_sys_bytes`
  - `readio_cloud_process_uptime_seconds`
  - `readio_cloud_sqlite_db_size_bytes`
  - `readio_cloud_sqlite_wal_size_bytes`
  - `readio_cloud_data_filesystem_free_bytes`
  - `readio_cloud_data_filesystem_total_bytes`
  - `readio_cloud_transcript_assets_size_bytes`
- Metric labels use closed low-cardinality enums for `route`, `provider`, `status_class`, `error_class`, `cache_status`, and `mode`.
- Metric labels must not include full URLs, query strings, route IDs, search terms, GUIDs, header names/values, raw errors, request/response bodies, audio bytes, or transcript text.

## 6. Log Access Contract

Cloud API logs remain stdout logs produced by Go `slog` and Docker's configured json-file logging driver. `/admin/logs` remains available and memory-only.

- 00C can ship sanitized application logs to Grafana Loki directly from `apps/cloud-api` without Grafana Alloy, Promtail, Docker log drivers, systemd journal reading, Docker container log reading, or another host log shipping agent.
- Loki shipping is disabled unless `READIO_GRAFANA_LOKI_URL`, `READIO_GRAFANA_LOKI_USER`, and `READIO_GRAFANA_LOKI_TOKEN` are all configured in the backend container.
- `READIO_GRAFANA_LOKI_URL` and `READIO_GRAFANA_LOKI_USER` are GitHub environment variables; `READIO_GRAFANA_LOKI_TOKEN` is a GitHub secret.
- Loki `env` labels are derived from backend `READIO_DEPLOY_ENV` first, then `VITE_GRAFANA_FARO_ENV` if unset. Values normalize to `production`, `preproduction`, or `unknown`.
- Optional backend-only knobs are `READIO_LOKI_LOG_LEVEL`, `READIO_LOKI_BATCH_SIZE`, `READIO_LOKI_FLUSH_INTERVAL_SECONDS`, and `READIO_LOKI_QUEUE_SIZE`.
- These Loki variables must stay out of `/env.js`, browser runtime config, and browser env allowlists.
- `/admin/logs` remains the same-origin, memory-only operator log surface. Loki is the durable remote application log surface when enabled.
- Docker logs remain available through `make logs-pre`, `make logs-prod`, or direct Docker operations on the VPS.
- 00B metrics and 00C logs are separate pipelines. OTLP metrics use `READIO_GRAFANA_OTLP_*`; Loki logs use `READIO_GRAFANA_LOKI_*`.
- Loki labels are exactly `service`, `env`, and `level`. Labels must not include request IDs, routes, upstream hosts, podcast IDs, episode IDs, GUIDs, search terms, user IDs, file paths, raw errors, IP addresses, user agents, tokens, API keys, cookies, or headers.
- Log lines may include sanitized fields such as timestamp, level, message, normalized route, status, elapsed milliseconds, error class, and request ID.
- The Loki queue is bounded and non-blocking for request paths. If the queue is full or the Loki endpoint fails, logs are dropped locally and requests continue. Shutdown performs a best-effort flush.

## 7. Faro Browser RUM Contract

Cloud UI Faro telemetry is optional and browser-public-config driven.

- `VITE_GRAFANA_FARO_URL` is required by the current deployment validation when Faro is configured for Grafana Cloud.
- `VITE_GRAFANA_FARO_SAMPLE_RATE` defaults to `0`; invalid or out-of-range values are safe.
- Faro collector URL, app name, env, and sample rate are public metadata, not credentials.
- No Grafana write credential, admin token, metrics token, relay secret, provider credential, Basic Auth value, local media, transcript text, or `/ops` session state may be read or exported by Faro.
- Sanitized schema diagnostics may include only schema name, provider enum, route class, issue path, issue code, and coarse error class.
- Faro initialization must fail open and never block app boot.

## 8. Host Observability Boundary

Grafana Cloud observability is app-owned OTLP metrics plus optional browser Faro telemetry.

- Do not install Grafana Alloy, node_exporter, local Prometheus, Loki agents, or any other persistent observability agent as part of 00A.
- Process/runtime, data filesystem, SQLite DB/WAL size, and configured transcript asset directory size are collected inside `apps/cloud-api` on a bounded interval. They are best-effort app-owned signals, not a full node_exporter replacement.
- `/admin/*` and `/ops` should not be publicly exposed without outer restrictions such as Cloudflare, nginx, VPN, or IP allowlists.
- Expected dashboards for 00A can cover upstream latency by provider/route, upstream errors by provider/error class, ASR relay request/error rates, and Faro crash/schema diagnostic rates.
- Cost controls require conservative Faro sampling, low-cardinality labels, and no raw log fields promoted to labels.
- Rollback clears Faro public config and/or unsets OTLP runtime variables while leaving `/ops` and `/admin/*` intact.
