## 1. 运行时拆分

- `apps/cloud-ui` 构建为静态前端产物
- `apps/cloud-api` 负责提供该产物并承载同源 API

## 2. 前端 Runtime Config

Cloud 前端的运行时配置通过后端 `/env.js` 发出。

- 浏览器可见字段必须保持在显式 allowlist 内
- 真实上游 secret 不得通过浏览器运行时配置暴露
- `apps/cloud-ui/public/env.js` 只应视为开发/默认模板

## 3. Discovery / Media 契约

- discovery 请求统一通过 Cloud 后端同源路由承载，包括 search / top / Editor's Picks batch lookup / podcast lookup / PI episode-list lookup
- 当前 Cloud 契约允许的情况下，媒体仍是浏览器直连优先
- backend fallback / proxy 的细节记录在 handoff 审计文档里

## 4. PWA 状态

Cloud 当前默认保持 Workbox / PWA 关闭，除非后续指令明确恢复。

## 5. 应用指标契约

配置 Grafana 环境变量后，`apps/cloud-api` 会通过 OTLP 直接向 Grafana Cloud 推送应用指标和应用侧采集的主机关键指标。

- 必需 OTLP 运行时配置是 `READIO_GRAFANA_OTLP_ENDPOINT`、`READIO_GRAFANA_OTLP_INSTANCE_ID` 和 `READIO_GRAFANA_OTLP_TOKEN`。
- Discovery CORS 由 `READIO_DISCOVERY_ALLOWED_ORIGINS` 归属；proxy CORS 仍由 `READIO_PROXY_ALLOWED_ORIGINS` 归属。
- 浏览器公开英文词典配置通过 `READIO_EN_DICTIONARY_API_URL` 与 `READIO_EN_DICTIONARY_API_TRANSPORT` 部署，并且只由后端生成的 `/env.js` 暴露。
- `READIO_GRAFANA_OTLP_TOKEN` 是仅后端的运行时配置；不得放入 `/env.js` 或浏览器 allowlist。
- 当前 00A 部署不暴露 Prometheus scrape endpoint，也不安装本地 Prometheus 或 Grafana Alloy。
- 当前应用指标包括：
  - `readio_cloud_http_request_duration_seconds`
  - `readio_cloud_upstream_request_duration_seconds`
  - `readio_cloud_upstream_errors_total`
  - `readio_cloud_asr_relay_requests_total`
- 当前应用侧主机指标包括：
  - `readio_cloud_process_goroutines`
  - `readio_cloud_process_heap_alloc_bytes`
  - `readio_cloud_process_memory_sys_bytes`
  - `readio_cloud_process_uptime_seconds`
  - `readio_cloud_sqlite_db_size_bytes`
  - `readio_cloud_sqlite_wal_size_bytes`
  - `readio_cloud_data_filesystem_free_bytes`
  - `readio_cloud_data_filesystem_total_bytes`
  - `readio_cloud_transcript_assets_size_bytes`
- Metric labels 使用封闭的低基数枚举：`route`、`provider`、`status_class`、`error_class`、`cache_status`、`mode`。
- Metric labels 不得包含完整 URL、query string、route ID、搜索词、GUID、header 名称/值、原始错误、请求/响应 body、音频字节或 transcript 文本。

## 6. 日志访问契约

Cloud API 日志继续由 Go `slog` 写入 stdout，并由 Docker 当前配置的 json-file logging driver 保存。`/admin/logs` 继续可用且只保存在内存中。

- 00C 可以由 `apps/cloud-api` 直接把脱敏后的应用日志推送到 Grafana Loki，不安装 Grafana Alloy、Promtail、Docker log driver、systemd journal 读取器、Docker container log 读取器或其他主机日志 shipping agent。
- 只有同时配置后端容器内的 `READIO_GRAFANA_LOKI_URL`、`READIO_GRAFANA_LOKI_USER` 和 `READIO_GRAFANA_LOKI_TOKEN` 时，Loki 投递才会启用。
- `READIO_GRAFANA_LOKI_URL` 和 `READIO_GRAFANA_LOKI_USER` 放在 GitHub environment variables；`READIO_GRAFANA_LOKI_TOKEN` 放在 GitHub secret。
- Loki `env` label 优先来自后端 `READIO_DEPLOY_ENV`，未设置时回退到 `VITE_GRAFANA_FARO_ENV`。取值会规范化为 `production`、`preproduction` 或 `unknown`。
- 可选的仅后端参数是 `READIO_LOKI_LOG_LEVEL`、`READIO_LOKI_BATCH_SIZE`、`READIO_LOKI_FLUSH_INTERVAL_SECONDS` 和 `READIO_LOKI_QUEUE_SIZE`。
- 这些 Loki 变量不得进入 `/env.js`、浏览器运行时配置或浏览器 env allowlist。
- `/admin/logs` 仍是 same-origin、memory-only 的运维日志界面。启用 Loki 后，Loki 是持久化远端应用日志界面。
- Docker logs 仍可通过 `make logs-pre`、`make logs-prod` 或 VPS 上的直接 Docker 操作查看。
- 00B 指标与 00C 日志是两条独立 pipeline。OTLP 指标使用 `READIO_GRAFANA_OTLP_*`；Loki 日志使用 `READIO_GRAFANA_LOKI_*`。
- Loki labels 只能是 `service`、`env` 和 `level`。Labels 不得包含 request ID、route、upstream host、podcast ID、episode ID、GUID、搜索词、user ID、文件路径、原始错误、IP、user agent、token、API key、cookie 或 header。
- 日志行可以包含脱敏后的 timestamp、level、message、规范化 route、status、elapsed milliseconds、error class 和 request ID。
- Loki 队列是有界的，并且不阻塞请求路径。队列满或 Loki endpoint 失败时会本地丢弃日志，请求继续执行。服务关闭时会进行 best-effort flush。

## 7. Faro 浏览器 RUM 契约

Cloud UI Faro telemetry 是可选能力，由浏览器公开配置驱动。

- 当前部署验证要求配置 `VITE_GRAFANA_FARO_URL`，用于接入 Grafana Cloud Faro。
- `VITE_GRAFANA_FARO_SAMPLE_RATE` 默认是 `0`；非法或越界值安全回退。
- Faro collector URL、app name、env 和 sample rate 是公开元数据，不是凭证。
- Faro 不得读取或导出 Grafana write credential、admin token、metrics token、relay secret、provider credential、Basic Auth value、本地媒体、transcript text 或 `/ops` session state。
- Sanitized schema diagnostics 只能包含 schema name、provider enum、route class、issue path、issue code 和粗粒度 error class。
- Faro 初始化必须 fail-open，不能阻塞 app 启动。

## 8. 主机可观测性边界

Grafana Cloud observability 由应用侧 OTLP 指标和可选的浏览器 Faro telemetry 组成。

- 00A 不安装 Grafana Alloy、node_exporter、本地 Prometheus、Loki agent 或任何其他持久化观测 agent。
- Process/runtime、data filesystem、SQLite DB/WAL size 以及配置的 transcript asset directory size 都由 `apps/cloud-api` 在有界 interval 内采集。这些是 best-effort 的应用侧主机信号，不是完整的 node_exporter 替代。
- `/admin/*` 与 `/ops` 不应在没有 Cloudflare、nginx、VPN 或 IP allowlist 等外围限制的情况下公开暴露。
- 00A 阶段的预期 dashboard 可以覆盖按 provider/route 的 upstream latency、按 provider/error class 的 upstream errors、ASR relay request/error rates，以及 Faro crash/schema diagnostic rates。
- 成本控制要求保守的 Faro sampling、低基数 labels，并且不把 raw log fields 提升为 labels。
- 回滚时清空 Faro public config 或取消 OTLP 运行时变量，同时保留 `/ops` 与 `/admin/*`。
